# MAX 26.22.2 GP V2 — Mods by JohNick

**Дата**: 2026-07-13 · **Сток**: 26.22.2 (build 6779) · **Android**: 8.0+ (API 26+)
**APK**: 4 файла (arm64 + arm7 × main + clone) · Universal (GP) · Подпись v2+v3

---

## 🐞 Что исправлено в V2

- **📢 Каналы = сток при включённых античиталках.** При `antiRead`/`smartAntiRead`/`invisible ON` каналы теперь ведут себя как в стоке: read-mark уходит на сервер без подавления. Раньше каналы попадали под те же WS-гейты, что и личные чаты — закрыл и снова открыл MAX, канал снова выглядел непрочитанным.

## 🐞 Что исправлено в V1 vs 26.22.1 V2

- **🔴 «Умная античиталка» — 3 бага стока 26.22.2.** На новом стоке при `smartAntiRead=ON`: (A) метка «прочитано» уходила сразу при открытии чата, а не после ответа; (B) при повторном открытии чата все входящие снова показывались непрочитанными; (C) чат не скроллился в конец при открытии. Все три chokepoint'а стока переехали одновременно (batch WS-send, реальный chat-open driver `qza.a`, event-holder `Lege`) — переязвали каждый.
- **🔄 Обычная `antiRead`** теперь тоже блокирует внутричатовый счётчик «N новых» и скроллит в конец при открытии чата. Раньше эта логика работала только при `smartAntiRead` — что нарушало антидетект обычной античиталки.
- **📁 `startOnNew` RE-ENABLED** — «Стартовая папка Новые» вернулась после DEFER в 26.22.0/1. Restore-guard тоже вернулся — при перезапуске приложения активная папка не перетирается на «Все».
- **🔎 «Проверить обновления»** — short-tap по строке версии (Настройки → О приложении) открывает диалог с текущей и последней доступной версией. **Автопроверки при старте НЕТ** — только по явному тапу. Long-tap по-прежнему открывает дев-меню.
- **🇷🇺 Баннер «Госуслуги» — реальный чокпоинт.** В 26.22.1 тумблер `hideDigitalId` прятал только пункт «Цифровой ID» из списка настроек, но hero-карточку в шапке — нет (она рендерится server-driven циклом по PMS-конфигу `0x9d`). Теперь скрывается и то, и другое одним тумблером.

## ⚠ Временно отключено в V2

- **`hideFavorites`** — тумблер в UI есть, но эффекта не даёт. В стоке 26.22.2 рефакторен chat-list combine-оператор (`b20.emit`); нужен fingerprint-resolver моделей. Вернём в 26.22.3+.

## ✅ Сохранено из 26.22.1 V2

Все функции 26.22.1 работают без изменений:
- `hidePromoWidgets` — скрытие серверных промо-виджетов в списке чатов.
- Анти-удаление переживает жёсткий kill приложения (OEM-фикс).
- Время удаления в метке «· удалено HH:MM».
- «Только чаты» (`hideChannels`) в кастомных папках.
- `gateOpcode5` — блокировка принудительного logout от сервера.
- Стабильные push на клон между сессиями.
- Отправка любых файлов с маскировкой `.bin`, Невидимка-каскад, реакции, скрытие «Пригласить/Найти» в футере.

## 📱 Установка

- Ставится поверх прошлой версии мода: `adb install -r <apk>` — данные сохраняются.
- Первая установка — разрешить неизвестные источники, войти по номеру.
- Клон — после установки один раз: Настройки → Приложения → MAX Clone → Очистить данные → войти по SMS.
- **Минимальный Android — 8.0** (API 26). На 5.0/6.0/7.x не установится — ограничение стока.
