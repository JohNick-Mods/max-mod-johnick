## 🆕 Новое в 26.23.2 RS V1

### 📦 Источник — RuStore single-file (2 APK вместо 4)
С 26.23.2 базовый источник — монолитный APK из RuStore (`MAX_(RS-L)_v.26.23.2(6779)`), уже содержащий все ABI. Пайплайн переведён под это:
- **2 универсальных APK** (`MAX_26.23.2_universal_V1.apk` + `_clone.apk`) вместо 4 раздельных arm64/arm7.
- По умолчанию вырезаны `lib/x86` + `lib/x86_64` (~56 МБ) — у русскоязычных пользователей x86-устройств почти нет. Итог ~50 МБ на APK.
- `emoji_sprite` в `resources.arsc` на месте → реакции работают без сторонней замены arsc.

### 🗑 Секунды и иконка корзины на удалённых — теперь безусловно
Маркер удалённого сообщения переработан и **больше не требует тумблера**:
- Время удалённого сообщения показывается с точностью до секунды: **`HH:mm:ss dd.MM`** (раньше — «· удалено HH:MM»).
- На удалённом сообщении рисуется иконка **🗑** (корзина) — визуально нагляднее текстовой метки.

Обе правки безусловны (работают на самом рендере). Набор «сохранённых удалённых» по-прежнему наполняется только при `antiDelete=ON` в момент удаления.

### 📁 «При открытии — папка Новые» (`startOnNew`) работает и в основном приложении
Раньше `startOnNew` открывал «Новые» только в клоне: в основном приложении сохранённый `folder_id` восстанавливался позже и перетирал наш выбор. Добавлен гейт восстановления папки в `ChatsTabWidget.onViewCreated` — при `startOnNew=ON` восстановление сохранённой папки зануляется, и «Новые» выживают. Проверено на устройстве в **основном и в клоне**.

### 🔢 Счётчик непрочитанных не накручивается на холодном старте
Известное ограничение 26.23.0 V8.1 («счётчик на иконке чата может накрутиться на удалённое/прочитанное сообщение») устранено: при `Невидимке` и «догнанном» read-ledger бейдж непрочитанных на холодном старте больше не возвращается для уже прочитанных чатов. Новое входящее двигает позицию выше снапшота → бейдж появляется штатно.

### 🎛 Dev-меню без нерабочего пункта «Дизайн система»
Из инженерного Dev-меню (long-press по версии) убран пункт **«Дизайн система»** — в стоке это `stub_release`-экран без контента (пустой чёрный экран). Остаются рабочие пункты.

### 🔧 R8-rebump под сток 26.23.2
Все apply-скрипты перерезолвены под новую R8-обфускацию стока 26.23.2 (сменившиеся обфусцированные имена классов/полей/методов). Проведена полная перекрёстная верификация ~118 патчей против собранного дерева (класс бага «silent no-op» — устаревший тип в дискриминаторе фильтра).

## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) — независимый тумблер |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно. Оба блокируют внутричатовый счётчик «N новых» и скроллят в конец при открытии.

### Анти-удаление (`antiDelete`) — 4 слоя + офлайн-персист
1. **L3 — Central WS chat-event**: собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear**: локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер 🗑 + время `HH:mm:ss dd.MM`** в строке времени (безусловно на рендере).
5. **Офлайн-удаления через FCM** — метка восстанавливается на следующем старте, если удаление пришло при убитом OEM-killer'ом приложении.

Восстановленные удалённые сохраняются с `isRead=1` — не задирают счётчик непрочитанных. Общий гейт `antiDelete`: **ON** → защита; **OFF** → поведение стока.

### 📢 Каналы = сток при включённых античиталках
При `antiRead`/`smartAntiRead`/`invisible` каналы ведут себя как в стоке: read-mark для каналов всегда уходит на сервер.

Все по умолчанию **OFF** (кроме `amoledTheme` — авто при тёмной теме) — до первого включения мод ведёт себя идентично стоку (антидетект). Диалог настроек следует системной теме. Открытие: **Настройки → О приложении → long-press по строке версии**.

## 🔒 Security-инварианты

### 🌐 DNS-блокировка телеметрии (P0)
```
trace-flow.ru
sdk-api.apptracer.ru
tracker-api.vk-analytics.ru
tracker.my.com
top-fwz1.mail.ru
```

### 🔕 Приваси-стабы (P0/P1)
- **myTracker (VK)** — вызовы SDK нейтрализованы (`return-void`).
- **Tracer-инициализаторы (7×)** — застаблены (`EMPTY_LIST` / `Object()`), SDK не стартует.
- **Firebase Installations** — `getComponents() → emptyList()` (нет HTTP к googleapis).
- **Фоновая выгрузка контактов** — метод нейтрализован (`return-void`).
- **NFC-HCE** — APDU-ответ `SW 6F00` (сервис не обслуживается).
- **TrustManager accept-all** — доверие «любому» сертификату заменено на `throw`.
- **No-IP телеметрия** — IP-адрес не отправляется в телеметрию соединения.

### ✍️ Подпись
- **apksigner**, схемы **v2 + v3** (без v1 / JAR-signing).
- Ключ: `debug.jks` (стандарт Mods by JohNick), cert **SHA-1 `d6a7d757…1298b`** — идентичен для всех сборок 26.x → `install -r` поверх любой предыдущей версии мода (включая 26.23.0 GP) без потери данных.

### 📦 Нативные библиотеки
Удаляется **только `libtracernative.so`**. Остальные `.so` — оригинальные из стока 26.23.2. Дополнительно вырезаны `lib/x86` + `lib/x86_64`.

## ⚠ Честно: чего НЕТ / ограничения в 26.23.2 RS V1

- **AMOLED — покрытие ~95%**: список чатов, окно чата и bottom-sheets могут остаться тёмно-серыми (рисуют собственный фон).
- **`#FILE-1`**: PDF / видео / m4a из основного аккаунта временами отправляются с проблемами — диагностика продолжается.
- **Тумблер `improveNotifications` скрыт из UI** (жёсткие проверки Android 15 / OxygenOS 15).
- **«Цифровой ID» и «Госуслуги баннер»** контролируются одним `hideDigitalId` (общий серверный ключ PmsProperty).
- **Требуется one-time `pm clear ru.oneme.ap2`** после установки APK клона.
- **x86/x86_64 не поддерживаются** (вырезаны для экономии размера) — для эмуляторов x86 нужна сборка `--full-abi`.
- **Минимальная версия Android — 8.0** (API 26) — ограничение стока.
