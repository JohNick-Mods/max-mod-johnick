# MAX 26.23.0 GP V8.1 — Mods by JohNick

**Дата**: 2026-07-24 · **Сток**: 26.23.0 (build 6777) · **Мин. Android**: 8.0 (API 26)

Точечный хотфикс поверх V8: устранён краш на смарт-часах при вводе номера в SMS-регистрации, и приведён в порядок ченжлог в диалоге проверки обновлений (полный текст + скролл вниз). Ставится через `install -r` поверх V8 — данные сохраняются.

## 🐞 Изменения V8.1

- **⌚ Краш на часах при вводе номера SMS — устранён.** Проходил под ART strict-verify на Android 15/16: класс OTP-адаптера отвергался (`VerifyError` в watch-ветке `vnf.w()`) — верификатор видел, что регистр `p1` после установки id-атрибута уже целочисленный, а фикс V7.1 читал его как Context. Теперь Context берётся из `p2` (сам View OTP-ячейки) — верификатор-безопасно, даже если watch-ветка недостижима на конкретном устройстве. На телефонах и планшетах поведение не меняется (сток байт-в-байт).
- **📜 Ченжлог в «Проверить обновления» — виден целиком, скроллится вниз.** Раньше текст релиза жёстко обрезался ~5-6 строк, а сам message в стоковом `AlertDialog` не всегда скроллился на темах Samsung One UI / Xiaomi. Теперь release-notes показываются целиком и обёрнуты в `ScrollView` — гарантированный вертикальный скролл в любой теме, кнопки «Установить» / «Отмена» остаются снизу.

## ✅ Сохранено из V8

- **🎨 «Цвет иконки»** — 6 цветов (Blue/Red/Green/Purple/Black/Orange), runtime-switch без переустановки.
- **📤 Порядок отправленных при «Невидимке»** — исправлен (пересортировка на входе адаптера, gated на `invisible=ON`).
- **🕹 «Невидимка» — единый мастер-тумблер** каскадно управляет `antiRead` / `antiTyping` / `antiPresence`.
- **🛡 Краш при запуске** (`startOnNew`) и **🛡 краш вкладки «Чаты»** (arsc animator) — устранены.

## ✅ Сохранено из V7.1

- **⌚ SMS-код на часах** — все 6 ячеек умещаются, последняя цифра принимается (ширина ячеек под watch-density; на телефонах без изменений).
- **🌑 AMOLED — авто при системной тёмной теме** (тумблер сохранён — можно вернуть штатную тёмно-серую).
- **📲 Push у клона — стабилен** (клон генерирует собственный device_id один раз).

## ✅ Сохранено из V7

- **🔕 Нет push об удалённых сообщениях при `antiDelete=ON`** — гейт в FCM-обработчике до запуска корутины.

## Установка

Ставится поверх V8 / V7.1 через `install -r` — данные сохраняются. Клон рядом с оригиналом.

- `MAX_26.23.0_arm64_V8.1.apk` — современные телефоны (2018+). **Если не уверены — этот.**
- `MAX_26.23.0_arm7_V8.1.apk` — старые 32-битные устройства.
- `MAX_26.23.0_arm64_V8.1_clone.apk` / `MAX_26.23.0_arm7_V8.1_clone.apk` — клон для второго аккаунта.

## Подпись и целостность

- Подпись **v2 + v3** (без v1). Cert SHA-1: `d6a7d757…1298b` — совместимо с `install -r` поверх любой 26.x сборки мода.
- `SHA256SUMS` в ассетах Release — контрольные суммы.
- Smoke-тест: Samsung S25, Android 16 — main + clone без FATAL / VerifyError; отдельно эмуляция часов 466×466 + density 320 — старый краш SMS-регистрации не воспроизводится, экран OTP открывается корректно.

## Известные ограничения 26.23.0 V8.1

- **Порядок отправленных при «Невидимке»** — исправлен в подавляющем большинстве случаев; в редких сценариях (задержка reconcile больше одного тика адаптера) отдельная пара сообщений может встать не по времени. Переоткрытие чата всегда возвращает правильный порядок.
- **Счётчик непрочитанных** на иконке чата может накрутиться на удалённое сообщение (сервер шлёт итоговое число без ID). Само сообщение видно в чате. *(запланировано в следующей версии)*
- **AMOLED** — покрытие ~95%: список чатов и всплывающие шторки могут остаться тёмно-серыми.
