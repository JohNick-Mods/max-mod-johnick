# MAX 26.23.0 GP V7 — Mods by JohNick

**Дата**: 2026-07-20 · **Сток**: 26.23.0 (build 6777) · **Мин. Android**: 8.0 (API 26)

Хотфикс поверх V6. Один точечный фикс: при активной «Антиудалялке» push-уведомления об удалённых собеседником сообщениях больше не приходят в шторку. Ставится через `install -r` без потери данных.

## 🐞 Фикс V7

- **🔕 Нет push об удалённых сообщениях при antiDelete=ON.** Раньше при активной «Антиудалялке» приходило уведомление даже о сообщении, которое собеседник удалил (мод его сохраняет в чате, но push всё равно всплывал в шторке). Теперь такие push подавляются.
  Механизм: гейт в FCM-обработчике (`FcmMessagingService`) ДО запуска корутины — достаётся `msgid` из push-payload; если `antiDelete=ON` и это сообщение помечено удалённым (в списке `deletedKept`), уведомление пропускается. Обычные push приходят как раньше. БД не изменяется.

> ⚙️ Реализовано через helper `AppFlags.isPushSuppressed(Bundle)`, вызываемый в точке приёма FCM (свободные регистры), — а не внутри корутины, чтобы не задеть её состояние. Любая ошибка разбора payload → push проходит штатно.

## Установка

Ставится поверх V6 через `install -r` — данные сохраняются. Клон рядом с оригиналом.

- `MAX_26.23.0_arm64_V7.apk` — современные телефоны (2018+). **Если не уверены — этот.**
- `MAX_26.23.0_arm7_V7.apk` — старые 32-битные устройства.
- `MAX_26.23.0_arm64_V7_clone.apk` / `MAX_26.23.0_arm7_V7_clone.apk` — клон для второго аккаунта.

## Известные ограничения 26.23.0 V7

- **Счётчик непрочитанных** на иконке чата может накрутиться на удалённое сообщение (сервер шлёт итоговое число без ID сообщения — точечно вычесть нельзя без отдельной доработки). Само сообщение видно в чате.
- **AMOLED** — покрытие ~95%: список чатов и всплывающие шторки могут остаться тёмно-серыми.
