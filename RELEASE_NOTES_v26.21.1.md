# MAX 26.21.1 — Mods by JohNick V2

Универсальная сборка (GP) · arm64 + arm7 · Main + Clone · 4 APK · build 6761

## Что нового в V2 (vs V1)

### 🌐 4 новых нейтрализации серверных транспортных рубильников

Сервер MAX может через WebSocket на лету менять «поведение» клиента: включать клиентский DoH (обход DNS-блокировки), отключать TLS-валидацию, перенаправлять на другой WS-узел, включать внутренний LOG-канал сериализации. Стоковые дефолты безопасны, но серверная активация происходит без ведома пользователя и без обновления клиента.

V2 закрывает 4 канала:

1. **`net-ssl-session-validate` → форс TRUE.** Ключ TLS-валидации сессии. Гейт в парсере `pmsMap` (`c2/lqk.smali::d`) игнорирует серверное значение и всегда возвращает `Boolean.TRUE`. Клиент всегда проверяет сертификат.
2. **`net-client-dns-enabled` → форс FALSE.** Ключ клиентского DoH-резолвера. Гейт в том же парсере всегда возвращает `Boolean.FALSE`. DNS-запросы MAX идут в системный резолвер — Lionheart DNS / AdGuard / NextDNS реально применяются к трафику.
3. **WS-редирект — заблокирован.** Серверная команда `SESSION_INIT.host/port/tls` игнорируется в `c3/de6.smali` — клиент остаётся на первоначальном WS-узле.
4. **WS-опкод 5 — короткое замыкание.** Служебный канал сериализации трафика на клиенте (внутренний LOG). Обработчик команд `c1/zwa.smali::j` коротко замыкает опкод 5 → `return-void`. Портировано из независимой сборки Geen MAX 26.20.2.

Все 4 нейтрализации срабатывают автоматически, без тумблеров — базовая приватность. Легитимные ~376 рубильников `pmsMap` работают штатно (`experiments`, `forceUpdateVersion` и др.).

Скрипты: `apply_v_pms_neutralize.py`, `apply_v_gate_opcode5.py`, `apply_v_p0_3_ws_yf.py` (реактивирован). Sentinel: `# :pms_blacklist_gate`, `# :gate_opcode5_dead_zone`, `# :redirect_host_guard`.

### 🔕 Тумблер #16 — «Скрыть чат "Полезные уведомления"»

Верифицированный промо-чат MAX теперь скрывается пользовательским тумблером. Категория «Сервисные чаты» — вместе с `hideVerifyCodes`, `hideMaxOfficial`, `hideInteresting`. Работает через тот же chokepoint `Lu00.b(List, Runnable)V`, что и остальные фильтры чатов из V1.

### 🎨 Тема диалога флагов следует системной

В V1 при системной тёмной теме диалог настроек мода отображался белым окном. В V2 диалог инициализируется с `Configuration.uiMode`:
- Тёмная тема системы → `Theme_DeviceDefault_Dialog_Alert`
- Светлая тема системы → `Theme_DeviceDefault_Light_Dialog_Alert`

Все под-диалоги категорий получили тот же theme-picker (правится в генераторе `gen_appflags.py`).

### 💡 Меню флагов открывается коротким тапом

В V2 short-tap по строке версии в настройках MAX открывает наше меню тумблеров (в V1 работал только long-press; long-press по-прежнему работает).

## Что не менялось vs V1

- Сток тот же (build 6761), R8-карта не менялась.
- SMS-fix при регистрации (VerGate), hotfix folder-hide через chokepoint `Lu00.b`, скрытый жест long-press → нативное dev-меню MAX (`:settings/dev`), always-on фичи (скриншоты, no-IP telemetry, автоповорот, AppLink prompt) — без изменений.

## Известные ограничения

- **«Отметить непрочитанным» в режиме Невидимка** откатывается серверным sync-путём (WRITE-side `Lgzh;->a(...)`). Требуется per-chat реестр `{chatId: timestamp}` — вынесено на следующий релиз.
- **Персист device_id клона при полном `pm clear`** — теряется, потребуется повторный вход по SMS. При штатном `install -r` сохраняется.

## Установка

**Обновление с любой 26.x от Mods by JohNick:** ставится поверх, данные сохраняются. Cert SHA-1 `d6a7d757…1298b` — одинаков. **Не удаляйте** предыдущую версию.

**Первая установка:** разрешить установку из неизвестных источников → открыть APK → «Установить» → войти по номеру.

**Если стоит оригинальный MAX:** удалить перед установкой основного мода (пакет совпадает). Клон можно ставить рядом с оригиналом.

## Цифровой ID — если открывается браузер

Внутренний фикс встроен. Из-за переподписи Android может не доверять автоматическому открытию ссылок `max.ru`. Лечится один раз:

- В приложении: попап «Цифровой ID» → «Открыть настройки» → «Открывать поддерживаемые ссылки»
- Или: **Настройки телефона → Приложения → MAX → «Открывать по умолчанию» → включить**
- Или через ADB:
  ```
  adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
  ```

Подробности — `DIGITAL_ID_README.md`.
