# MAX 26.21.1 — Mods by JohNick V1

Универсальная сборка (GP) · arm64 + arm7 · Main + Clone · 4 APK · build 6761

## Что нового (vs 26.21.0)

### Hotfix — скрытие чатов «Избранное» / «Коды подтверждения» / «MAX» в списке чатов

В 26.21.0 тумблеры `hideFavorites` / `hideVerifyCodes` / `hideMaxOfficial` действовали только на верхние вкладки-папки, а сами чаты в списке на вкладке «Все» оставались видны. В 26.21.1 фильтр работает и на самом списке чатов:

- **Истинный chokepoint AsyncListDiffer — `Lu00.b(List, Runnable)V`.** Класс `Ljq8` (базовый `ListAdapter`) — не единственный источник submit-путей: около десяти direct-callsites вызывают `Lu00.b` напрямую, обходя обёртки `Ljq8.H/I`. Инъекция в `Lu00.b` — единственный универсальный узел, покрывающий все пути AsyncListDiffer.submitList независимо от адаптера.
- **Модель UI recent-chats — `Lf03`, а не `Ly66`.** Ly66 — master-index (эмит-путь), Lf03 — визуальная UI-модель списка. `filterChatList` теперь two-model: title=`e:CharSequence` для Ly66 / title=`c:CharSequence` для Lf03.
- **Официальный чат «MAX» использует смешанный омоглиф-набор.** Буквы M/A/X внешне идентичны кириллическим М/А/Х — сервер может отдавать любую из 2³=8 комбинаций. Фильтр перебирает все восемь через `equals`.

Проверено на устройстве (Samsung S25): при включённых тумблерах Избранное / Коды подтверждения / MAX исчезают из списка на вкладке «Все».

### Что не менялось vs 26.21.0

Сток тот же (build 6761), R8-карта не менялась. 15 тумблеров, скрытый жест long-press по номеру версии → нативное dev-меню MAX, always-on фичи (скриншоты, no-IP telemetry, автоповорот, AppLink prompt), 13/13 нейтрализованных угроз — без изменений.

### Известное ограничение

В режиме Невидимка ручная пометка чата непрочитанным через контекстное меню откатывается серверным sync-путём. Требуется отдельный per-chat реестр `{chatId: timestamp}` с TTL — вынесено на следующий релиз.

### Убрано из пайплайна

Три неработавших шага (`apply_v72_hide_mm2_list`, `apply_v73_g93_ctor_filter`, `apply_v74_jq8_setlist_filter`) — не попадали на нужный путь. Заменены единым `apply_v75_u00_submit_filter` (истинный chokepoint).

## Установка

**Обновление с любой 26.x от Mods by JohNick:** ставится поверх, данные сохраняются. Cert SHA-1 `d6a7d757…1298b` — одинаков. **Не удаляйте** предыдущую версию.

**Первая установка:** разрешить установку из неизвестных источников → открыть APK → «Установить» → войти по номеру.

**Если стоит оригинальный MAX:** удалить перед установкой основного мода (пакет совпадает). Клон можно ставить рядом с оригиналом.

## Цифровой ID — если открывается браузер

Внутренний фикс встроен — обычно Цифровой ID создаётся сразу. Из-за переподписи Android может не доверять автоматическому открытию ссылок `max.ru`. Лечится один раз:

- В приложении: попап «Цифровой ID» → «Открыть настройки» → «Открывать поддерживаемые ссылки»
- Или: **Настройки телефона → Приложения → MAX → «Открывать по умолчанию» → включить**
- Или через ADB:
  ```
  adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
  ```

Подробности — `DIGITAL_ID_README.md`.
