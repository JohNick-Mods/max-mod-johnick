## 🐞 Исправлено в V2

### 🔴 Crash-loop `improveNotifications` (OxygenOS 15)
На OnePlus 13/15 (OxygenOS 15 / Android 15) при включении тумблера «Улучшить уведомления» приложение мгновенно падало и не запускалось до перевода в режим «Самолёт» → отключения тумблера. Причина: даже `IMPORTANCE_LOW` (`0x2`) для foreground-канала реджектится системой (`CannotPostForegroundServiceNotificationException`) → крах → флаг остаётся ON в `SharedPreferences` → петля.

**Фикс**: тумблер **скрыт из UI** (не появляется в диалоге настроек); шаг создания отдельного канала `.bg` **отключён** в пайплайне сборки. Даже если у пользователя флаг `improveNotifications=ON` остался в конфиге предыдущей версии, канал `.bg` не создаётся, `BackgroundListenService` продолжает работать на стоковом канале `.misc` — крашей нет. Поле `improveNotifications:Z` в `AppFlags` сохранено — для двоичной совместимости с классами `KeepAliveManager` / `NetworkChangeCallback`, читающими `sget-boolean`. Фича вернётся в 26.23.x после device-matrix smoke на OxygenOS/HyperOS/ColorOS с `IMPORTANCE_DEFAULT` (0x3).

### 🆕 Тумблер `hidePromoWidgets` — скрытие серверных промо-баннеров в списке чатов
На отдельных аккаунтах в списке чатов появлялся баннер «MAX для бизнеса» с кнопкой «Открыть» (между обычными чатами). Это не чат, а серверный promo-виджет, поступающий из PMS-конфига по ключу `chats-list-promo-link-enabled`. Стандартный фильтр списка чатов (`filterChatList`) его не ловил — виджет приходит отдельным типом модели.

**Фикс**: новый тумблер (категория «Чаты», OFF-default) — при включении удаляет ключ `chats-list-promo-link-enabled` из карты серверного конфига в момент её сохранения в PMS-обёртку `Lxs5.<init>(Map)`. Getter возвращает null → баннер не рендерится. Патч также покрывает все будущие промо-виджеты, приходящие под тем же ключом.

## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно — включение одного снимает другой.

### Анти-удаление (`antiDelete`) — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event**: live-канал, собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear**: локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер «· удалено»** в строке времени + **время удаления** (`dk_times`).
5. **Офлайн-удаления через FCM** — если удаление пришло, пока приложение убито OEM-killer'ом (RealMe/ColorOS/MIUI: «Очистить всё», «Force stop», hard-kill), метка восстанавливается при следующем старте (msgId заносится в персистентный список на входе FCM-пуша `MessageRemoved`).

Общий гейт `antiDelete`: **ON** → защита; **OFF** → поведение стока.

## 🆕 Новое в 26.22.1 (относительно 26.22.0 V1)

**Скрытие:**
- **Скрыт промо-баннер «Госуслуги»** — в 26.22.0 `hideDigitalId` убирал только пункт «Цифровой ID» из списка Настроек, но промо-баннер вверху экрана оставался. Теперь при `hideDigitalId=ON` баннер тоже уходит.
- **`hideChannels` работает в кастомных папках** — в 26.22.0 каналы скрывались только в главном списке чатов; в пользовательских папках оставались. Добавлен второй chokepoint фильтрации (folder-scoped список).
- **`hidePromoWidgets` 🆕 (V2)** — скрытие серверных промо-виджетов в списке чатов (баннер «MAX для бизнеса» и подобные).

**Анти-удаление:**
- **Метка «удалено» переживает жёсткий kill приложения** — на OEM-прошивках (RealMe/ColorOS/MIUI и т.п.), где системный «Очистить всё» или Force Stop убивал приложение до записи метки. Теперь удаления, пришедшие через фоновый FCM-push (`MessageRemoved`), восстанавливаются при следующем старте.
- **Время удаления** в метке (не только сам факт).

**Транспорт (под тумблером OFF-default):**
- **`gateOpcode5`** — блокирует принудительный logout от сервера. При получении WS-команды «сбросить все задачи» (opcode 5) клиент её игнорирует, сохраняя сессию.

**Стабильность:**
- Push на клон (`ru.oneme.ap2`) стабильно приходят между сессиями — устранена ротация device_id, которая в V1 26.22.0 могла обнулять push-endpoint при нескольких холодных стартах. После установки клона по-прежнему требуется одноразовый `Настройки → Приложения → MAX Clone → Очистить данные`, дальше не нужен.
- SMS-регистрация нового номера на клоне работает штатно (устранён боковой эффект защитного гейта).

## ⚠ Честно: чего НЕТ в 26.22.1 V2

- **Тумблер `improveNotifications` временно недоступен** (скрыт из UI). Причина крайних крашей — жёсткие проверки Android 15 / OxygenOS 15 на IMPORTANCE фонового канала. Вернём в 26.23.x после device-matrix smoke.
- **2 транспортные нейтрализации остаются DEFERRED** до 26.23.x:
  - `WS-redirect` — используется штатно для sharding SMS-регистрации по диапазону номера; безусловный skip ломает вход по новым номерам. В 26.23.x — под тумблер OFF-default с UI-предупреждением.
  - `vmk` (SSL-session-validate / DNS-client-config) — реестр `PmsProperty` изменил тип default'ов, тривиальная замена вызывает `ClassCast`. Требует передекодировки getter'ов.
- **Требуется one-time `pm clear ru.oneme.ap2`** после установки APK клона (штатный шаг установки клона).
