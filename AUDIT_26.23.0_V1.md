## 🆕 Новое в 26.23.0 V1

### 🌑 AMOLED-тема (тумблер `amoledTheme`)

Чисто чёрный фон `#000000` для окна, status-bar и navigation-bar в тёмной теме — экономит батарею на OLED-панелях. Категория «Звонки и система», по умолчанию **OFF**.

**Гейт на системный dark-mode**: тема применяется **автоматически только когда система в тёмной теме**. При светлой системной теме тумблер не оказывает никакого эффекта (не трогает ни один цвет) — переключение на тёмную сразу активирует чёрный фон.

> **Известное ограничение (покрытие ~95%)**: список чатов, окно чата и всплывающие шторки снизу (bottom-sheets) рисуют собственный фон и не наследуют цвет окна — могут остаться тёмно-серыми. Заголовки, настройки, диалоги и полосы системы чернеют полностью.

### ⌚ Watch-mode — авто-адаптация под смарт-часы

Авто-подгонка интерфейса под Huawei Watch 3/4 Pro и аналогичные (1.43″, 466×466 px @ 326dpi). При старте приложение измеряет ширину экрана: если **меньше 500 px** — принудительно выставляется `density=240dpi` (hdpi). Layout мессенджера перестаёт вылезать за края круглого экрана, все элементы влезают. Локализация интерфейса при этом сохраняется.

**Тумблера нет** — режим определяется автоматически по ширине экрана. На телефонах (ширина заведомо >500 px) не срабатывает, поведение стоковое. Актуально прежде всего для сборки **arm7** (носимые устройства почти всегда 32-битные).

> **Trade-off**: на 1.43″ элементы визуально мельче — неизбежная плата за то, чтобы layout уместился целиком.

### 🔓 Антиприсутствие отвязано от «Невидимки»

Тумблер `antiPresence` стал **независимым** флагом. Раньше он взводился только как дочерний внутри мастер-переключателя «Невидимка» (`invisible`); теперь скрывать статус «в сети» / last-seen можно, не включая остальные приватные подфлаги. Мастер-переключатель по-прежнему взводит его вместе с остальными.

### 🎛️ Дев-меню — long-press по версии

Long-press по строке версии в **Настройки → О приложении** открывает нативное меню разработчика MAX. Dispatcher-класс переехал в стоке (`zh2` → `ri2`), Test-crash пункт переякорен (`i81` → `a91`) — переанкорено под 26.23.0.

### 🔎 Проверка обновлений — short-tap по версии

Short-tap по строке версии открывает `UpdateChecker`: сверка текущей версии с GitHub Release, кнопка перехода/скачивания-установки новой сборки. Dispatcher переехал в стоке (`xc` → `gd`). Проверка идёт через отдельный mirror-репозиторий (URL зашит в APK).

> **Автопроверка НЕ включена**: никаких фоновых сетевых запросов при старте — только по явному тапу.

## 🐞 Исправлено в 26.23.0 V1

### 🔴 Краш «умной античиталки» при скролле к низу чата

**Симптом**: на новом стоке чат не открывался вовсе — падение при попытке скролла к последнему сообщению.

**Корневая причина**: `NoSuchFieldError` — в стоке уехал field `Ljje;->g:J` в scroll-обработчике (`tea.c`), прежний якорь ссылался на несуществующее поле → исключение при инициализации open-flow.

**Фикс**: scroll-обработчик переякорен под новое имя поля в 26.23.0. Открытие чата с `smartAntiRead=ON`/`antiRead=ON` восстановлено, скролл в конец работает.

### 🙈 `hideInviteFriends` / `hideBusinessPromo` снова скрываются

**Симптом**: пункты «Пригласить друзей» и «MAX для бизнеса» перестали прятаться при включённых тумблерах.

**Корневая причина**: сменились res-id строк в стоке 26.23.0 (`0x7f110a2a`→`0x7f110a41` и `0x7f110a0c`→`0x7f110a12`). Гейты ссылались на старые id → не находили целевые пункты.

**Фикс**: обновлены res-id в обоих патчах. Скрытие работает под тумблерами `hideInviteFriends` и `hideBusinessPromo`.

### 📬 `antiDelete` isRead — удалённые больше не висят непрочитанными

**Симптом**: восстановленные анти-удалением сообщения задирали счётчик непрочитанных — бейдж показывал непрочитанные, которых визуально не видно.

**Корневая причина**: восстановленное удалённое сообщение сохранялось с `isRead=0`, попадая в подсчёт непрочитанных бейджа.

**Фикс** (`apply_v_antidel_isread_fix.py`): `iput isRead=1` в `mq2.smali` при восстановлении удалённого сообщения — метка «прочитано» проставляется, бейдж больше не растёт от анти-удалённых (`#ANTIDEL-1`).

### 🔧 R8-rebump под сток 26.23.0

Сток бампнут VK (build 6773→6777, versionName 26.23.0). **53 apply-скрипта** перерезолвены под новую R8-обфускацию. SMS-регистрация: User-Agent обновлён до 26.23.0 (vercode 6777) — код при регистрации приходит.

## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) — **26.23.0: независимый тумблер** |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно — включение одного снимает другой. И `antiRead`, и `smartAntiRead` одинаково блокируют внутричатовый счётчик «N новых» и скроллят в конец при открытии.

### Анти-удаление (`antiDelete`) — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event**: live-канал, собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear**: локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер «· удалено»** в строке времени + **время удаления** (`dk_times`).
5. **Офлайн-удаления через FCM** — если удаление пришло, пока приложение убито OEM-killer'ом (Realme/ColorOS/MIUI/Huawei: «Очистить всё», «Force stop», hard-kill), метка восстанавливается при следующем старте.

**26.23.0**: восстановленные удалённые сообщения сохраняются с `isRead=1` — не задирают счётчик непрочитанных бейджа.

Общий гейт `antiDelete`: **ON** → защита; **OFF** → поведение стока.

### 📢 Каналы = сток при включённых античиталках
При включённых `antiRead`/`smartAntiRead`/`invisible` каналы ведут себя как в стоке: read-mark для каналов всегда уходит на сервер, бейдж пропадает и не возвращается после cold-start. Античиталки продолжают работать без изменений для обычных чатов и групп.

## 🔒 Security-инварианты

### 🌐 DNS-блокировка телеметрии (P0)
Домены трекинга и телеметрии заблокированы на уровне DNS-резолвинга:

```
trace-flow.ru
sdk-api.apptracer.ru
tracker-api.vk-analytics.ru
tracker.my.com
top-fwz1.mail.ru
```

### 🔕 Приваси-стабы (P0/P1)
- **myTracker (VK)** — вызовы SDK нейтрализованы (`return-void`).
- **Фоновая выгрузка контактов** — метод нейтрализован (`return-void`).
- **NFC-HCE** — APDU-ответ `SW 6F00` (сервис не обслуживается).
- **TrustManager accept-all** — доверие «любому» сертификату заменено на `throw` (кастомный accept-all TM отклоняется).
- **No-IP телеметрия** — IP-адрес не отправляется в телеметрию соединения.

### ✍️ Подпись
- **apksigner**, схемы **v2 + v3** (без v1 / JAR-signing).
- Ключ: `debug.jks` (стандарт Mods by JohNick), cert **SHA-1 `d6a7d757…1298b`** — идентичен для всех сборок 26.x → `install -r` поверх любой предыдущей версии мода без потери данных.

### 📦 Нативные библиотеки
Из нативных либ удаляется **только `libtracernative.so`** (нативный tracer). Остальные `.so` — оригинальные из стока 26.23.0.

## ⚠ Честно: чего НЕТ / ограничения в 26.23.0 V1

- **AMOLED — покрытие ~95%**: список чатов, окно чата и bottom-sheets рисуют собственный фон, могут остаться тёмно-серыми.
- **Watch-mode**: на 1.43″ элементы визуально мельче (неизбежно при принудительном hdpi).
- **`#FILE-1`**: PDF / видео / m4a из основного аккаунта временами отправляются с проблемами — диагностика продолжается.
- **Тумблер `improveNotifications` временно скрыт из UI** (жёсткие проверки Android 15 / OxygenOS 15 на IMPORTANCE фонового канала).
- **2 транспортные нейтрализации остаются DEFERRED**:
  - `WS-redirect` — используется штатно для sharding SMS-регистрации; безусловный skip ломает вход по новым номерам.
  - `vmk` (SSL-session-validate / DNS-client-config) — реестр `PmsProperty` изменил тип default'ов, тривиальная замена вызывает `ClassCast`.
- **Разделение тумблеров «Цифровой ID» и «Госуслуги баннер»** — оба контролируются одним `hideDigitalId` (общий серверный ключ PmsProperty).
- **Требуется one-time `pm clear ru.oneme.ap2`** после установки APK клона (штатный шаг установки клона).
- **Минимальная версия Android — 8.0** (API 26). На более старых установка сорвётся с «Сбой при разборе пакета» — ограничение стока, не мода.
