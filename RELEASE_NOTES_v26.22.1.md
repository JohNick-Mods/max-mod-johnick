# MAX 26.22.1 V1 — Mods by JohNick

Релиз поверх стока **26.22.1 (build 6772)**. Приватность, нейтрализация телеметрии, UX-функции. **29 тумблеров**, все по умолчанию OFF (антидетект).

## 🆕 Новое в этой версии (относительно V1 26.22.0)

- **🛡️ Анти-удаление переживает жёсткий kill приложения.** На OEM-прошивках (MIUI/ColorOS/RealMe и т.п.) системная кнопка «Очистить всё» или «Остановить принудительно» раньше убивали приложение до записи метки — теперь удаления, приходящие через фоновый push, гарантированно попадают в персистентный список.
- **⏱️ Время удаления в метке** — «· удалено HH:MM» вместо просто «· удалено».
- **🇷🇺 Скрыт промо-баннер «Госуслуги»** на главной странице Настроек (тумблер `hideDigitalId` теперь убирает и баннер, не только пункт списка).
- **📁 «Только чаты» (hideChannels) работает в кастомных папках** — раньше каналы скрывались только в главном списке, теперь и в пользовательских папках.
- **🔔 Новый тумблер `improveNotifications`** — фоновая служба уходит на отдельный тихий канал (IMPORTANCE_LOW). Раньше foreground-notif и push-сообщения делили один канал → системное глушение фонового канала глушило и сообщения. При ON — раздельно.
- **🚪 Новый тумблер `gateOpcode5`** — блокирует принудительный logout от сервера (WS-опкод 5). При ON сессия сохраняется.
- **📱 Push на клон стабильнее** — устранена ротация device_id при холодных стартах. После установки APK клона нужен один раз `Настройки → Приложения → MAX Clone → Очистить данные`, далее не требуется.
- **📞 SMS-регистрация нового номера на клоне** — устранена регрессия из промежуточных сборок.

## 🐞 Исправлено

- **`improveNotifications` крах на Android 12+** — канал переведён с `IMPORTANCE_MIN` на `IMPORTANCE_LOW` (система реджектит foreground-notification на MIN → `CannotPostForegroundServiceNotificationException`).
- **NPE-защита в FCM-хендлере анти-удаления** — добавлен null-guard, чтобы повреждённый push не роняет обработчик.

## ⚠ Честно

- **2 транспортные нейтрализации по-прежнему отложены до 26.23.x:**
  - **WS-redirect guard** — используется штатно для sharding SMS-регистрации, безусловный skip ломает вход по новым номерам. Планируется под тумблер OFF-default + UI-warning.
  - **vmk (net-ssl-session-validate / net-client-dns-enabled)** — реестр `PmsProperty` изменил тип default'ов, тривиальная замена вызывает `ClassCast`. Требует передекодировки getter'ов.
- Базовые дефолты стока в этих точках безопасны (TLS-валидация включена, DoH отключен).
- Полный `pm clear` клона теряет device_id для push — потребуется повторный вход по SMS.

## Установка

`install -r` поверх любой версии мода 26.x — данные сохраняются (та же debug-подпись). Первая установка: разрешить «неизвестные источники» → открыть APK → войти по номеру.

| APK | Кому |
|---|---|
| `MAX_26.22.1_arm64_V1.apk` | большинство телефонов (с 2018 г.) |
| `MAX_26.22.1_arm7_V1.apk` | старые 32-битные |
| `MAX_26.22.1_arm64_V1_clone.apk` | клон (2-й аккаунт), пакет `ru.oneme.ap2` |
| `MAX_26.22.1_arm7_V1_clone.apk` | клон для 32-битных |

Полный технический разбор — [`AUDIT_26.22.1_V1.pdf`](AUDIT_26.22.1_V1.pdf).
