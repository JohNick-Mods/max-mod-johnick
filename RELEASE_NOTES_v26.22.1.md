# MAX 26.22.1 V2 — Mods by JohNick

Релиз поверх стока **26.22.1 (build 6772)**. Приватность, нейтрализация телеметрии, UX-функции. **29 тумблеров в UI**, все по умолчанию OFF (антидетект).

> **V2 hotfix** — устраняет crash-loop при `improveNotifications=ON` на OxygenOS 15 (OnePlus 13/15) и добавляет тумблер скрытия серверных промо-баннеров в списке чатов.

## 🐞 Исправлено в V2

- **🔴 Краш при включении «Улучшить уведомления» на OnePlus 13/15 (OxygenOS 15).** У части пользователей тумблер `improveNotifications=ON` приводил к `CannotPostForegroundServiceNotificationException` → приложение падало сразу и не запускалось до перевода в режим «Самолёт» и отключения тумблера. **Фикс:** тумблер скрыт из UI (не появляется в диалоге настроек); шаг создания отдельного канала `.bg` отключён — фоновая служба работает на стоковом канале. Даже если флаг остался ON в конфиге предыдущей версии, крашей больше нет. Вернём фичу в 26.23.x после device-matrix smoke на OxygenOS/HyperOS/ColorOS.

## 🆕 Новое в V2

- **📛 Тумблер «Скрыть промо-баннеры»** (`hidePromoWidgets`) — убирает серверные promo-виджеты из списка чатов. В том числе баннер **«MAX для бизнеса»** с кнопкой **«Открыть»**, встраиваемый между обычными чатами. Работает на уровне PMS-конфига: удаляет ключ `chats-list-promo-link-enabled` из карты флагов при её сохранении в обёртку `Lxs5.<init>(Map)` → getter возвращает null → баннер не рендерится. Покроет и будущие промо-виджеты, приходящие под тем же ключом.

## 🆕 Из V1 26.22.1 (относительно V1 26.22.0) — сохранено

- **🛡️ Анти-удаление переживает жёсткий kill приложения.** На OEM-прошивках (MIUI/ColorOS/RealMe и т.п.) удаления, приходящие через фоновый push, теперь гарантированно попадают в персистентный список.
- **⏱️ Время удаления в метке** — «· удалено HH:MM» вместо просто «· удалено».
- **🇷🇺 Скрыт промо-баннер «Госуслуги»** на главной странице Настроек (тумблер `hideDigitalId` теперь убирает и баннер).
- **📁 «Только чаты» (hideChannels) работает в кастомных папках** — добавлен второй chokepoint фильтрации.
- **🚪 Тумблер `gateOpcode5`** — блокирует принудительный logout от сервера (WS-опкод 5). При ON сессия сохраняется.
- **📱 Push на клон стабильнее** — устранена ротация device_id при холодных стартах. После установки клона нужен один раз `Настройки → Приложения → MAX Clone → Очистить данные`, дальше не требуется.
- **📞 SMS-регистрация нового номера на клоне** — устранена регрессия из промежуточных сборок.

## ⚠ Честно

- **Тумблер `improveNotifications` временно недоступен** (скрыт из UI). Вернётся в 26.23.x с IMPORTANCE_DEFAULT + device-matrix smoke.
- **2 транспортные нейтрализации по-прежнему отложены до 26.23.x:**
  - **WS-redirect guard** — используется штатно для sharding SMS-регистрации, безусловный skip ломает вход по новым номерам. Планируется под тумблер OFF-default + UI-warning.
  - **vmk** (net-ssl-session-validate / net-client-dns-enabled) — реестр `PmsProperty` изменил тип default'ов, тривиальная замена вызывает `ClassCast`. Требует передекодировки getter'ов.
- Базовые дефолты стока в этих точках безопасны (TLS-валидация включена, DoH отключен).
- Полный `pm clear` клона теряет device_id для push — потребуется повторный вход по SMS.

## Установка

`install -r` поверх любой версии мода 26.x — данные сохраняются (та же debug-подпись). Первая установка: разрешить «неизвестные источники» → открыть APK → войти по номеру.

| APK | Кому |
|---|---|
| `MAX_26.22.1_arm64_V2.apk` | большинство телефонов (с 2018 г.) |
| `MAX_26.22.1_arm7_V2.apk` | старые 32-битные |
| `MAX_26.22.1_arm64_V2_clone.apk` | клон (2-й аккаунт), пакет `ru.oneme.ap2` |
| `MAX_26.22.1_arm7_V2_clone.apk` | клон для 32-битных |

Полный технический разбор — [`AUDIT_26.22.1_V2.pdf`](AUDIT_26.22.1_V2.pdf).
