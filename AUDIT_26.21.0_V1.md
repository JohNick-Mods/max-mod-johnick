# ЧАСТЬ 1. Что нового относительно 26.20.2

## 1.1 Платформа

Версия — полный перенос мода на **свежий сток MAX 26.21.0 (build 6761)**. Все нейтрализации и пользовательские функции портированы по обновлённой R8-карте: при ребилде разработчик снова переименовал часть обфусцированных классов, поэтому каждая точка патча сверялась по содержимому (сигнатуры, строковые литералы, якоря), а не по имени. Ключевые смещения имён 26.20.2 → 26.21.0:

| Логика | 26.20.2 | 26.21.0 |
|---|---|---|
| VPN-детектор `hasTransport(0x4)` | `f34.e()Z` | `m44.c()Z` |
| VPN-плашка «Отключите ВПН» | `boi.a()Z` | `uti.a()Z` |
| Settings UI-хук (Toolbar) | `Lfwb` | `Lmyb` |
| FileProvider dynamic authority | `ze6.<init>` | `yg6.<init>` |
| Стек папок ViewPager2 (`hideSferum` полоска) | `js6.M` (адаптер страниц) | `rf3.pswitch_2b5` (источник — до `K.g` и `js6.M`) |
| Точка `startOnNew` | `pe3.pswitch_2b5` | `rf3.pswitch_2b5` (после `js6.M`, ViewPager2-виджет) |
| Точка `hidePhoneNumber` | `wb4.q()` | `Ld74.w()J` (`wb4.q()` в 26.21.0 мёртв) |

Особое: в 26.21.0 у `Toolbar` (`Lmyb`) удалён метод `getTitle()TextView` — вместо этого добавлено публичное поле `h:Landroid/widget/TextView;`. UI-хук перестроен на прямое чтение поля.

## 1.2 Новые и доработанные функции

### Новые тумблеры и функции с UI

- **Новый 16-й переключатель — «Скрыть папку "Избранное"» (`hideFavorites`).** В 26.20.2 было 15 переключателей. Скрывает вкладку-папку «Избранное» в списке чатов через ту же общую точку фильтрации, что и `hideAll` / `hideSferum` / `hideBusinessPromo` (`filterPageList` по подстроке в заголовке). По умолчанию **выключен**.
- **Скрытый жест — long-press по номеру версии → нативное dev-меню MAX.** В стоке 26.21.0 у самого MAX уже есть штатная фича: долгий тап по строке версии в настройках открывает нативный экран `:settings/dev` (внутренние дебаг-настройки), но она **загейчена под dev-канал сборки** (`c2/lh2.smali :pswitch_ae`: `if-ne p1, 0x3, :cond_f5` — если сборка не dev, ветка не срабатывает). Мод снимает гейт (сравнение регистра с самим собой — `if-ne v0, v0, :cond_f5` → переход НЕ берётся никогда → навигация выполняется всегда). Навигация идёт штатным роутером MAX (`Lrv4;->c`), без DI/NPE, `DevMenuScreen` инстанцируется самим приложением. Жест скрытый (обычный юзер не наткнётся), гейт-под-тумблер не нужен. Sentinel: `# :devmenu_version_gate_bypass`. Отличается от тумблера `debugMenu` (тот разблокирует Debug-меню звонков — совсем другой экран).

### Правки существующих фич

- **`hideSferum` — вкладка «Сферум» теперь скрывается в полоске вкладок.** В 26.20.2 фильтр стоял в `js6.M` (адаптер страниц ViewPager2) — прятал страницы, но не полоску вкладок (tab-strip). В 26.21.0 фильтр перенесён в источник — `rf3.pswitch_2b5` сразу после `check-cast v0,List`, **до** обеих точек-потребителей (`K.g` — tab-strip и `js6.M` — pages). Теперь скрывается и сама вкладка.
- **`hideSferum` — пункт «Войти в Cферум» теперь скрывается в настройках.** Список настроек в coroutine `c3/ngf.smali::invokeSuspend` пополняется в нескольких местах; в 26.20.2 наш фильтр стоял после одного из промежуточных `addAll` и не видел поздних вставок. В 26.21.0 фильтр перенесён на **финальную эмиссию списка** — перед `Laag;->k(null, v8)` (единственная точка выдачи готового списка). Ловит элементы независимо от блока-источника, включая server-driven пункт «Войти в Cферум» (`Lbkf → c:Lcbh → Lbbh → c:CharSequence`, поиск по подстроке `"ферум"` после `toLowerCase()` — ловит и латинский C, и кириллический С).
- **`hidePhoneNumber` — правильная точка для стока 26.21.0.** В прошлых версиях мода гейт стоял в `wb4.q()`, но в 26.21.0 разработчик перенёс логику — `wb4.q()` больше не вызывается для отображения номера в профиле. Гейт перенесён в `Ld74.w()J` (реальный геттер `serverPhone`, покрывает 40+ callsites отображения). При включённом `hidePhoneNumber` метод возвращает `0L`, и во всех местах отображения номер скрыт. Смежное поле `Ls84.a:J` (`serverId` / `userId`) — **не трогается** (используется 20+ core-классами внутренней маршрутизации, гейт по нему сломал бы приложение).
- **`startOnNew` — реактивирован под новый folder-стек.** В 26.21.0 подсистема папок переписана (наблюдатель списка папок теперь `rf3.pswitch_2b5` после `js6.M`, модель вкладки — `Ljq6`). `selectStartFolder(list, widget)` вызывается после `Lnq6;->N` — находит `Ljq6` с заголовком, начинающимся на «Новые», и выбирает вкладку через `widget.r1().h(index, false)` (ViewPager2 wrapper `Ljri`). Once-guard `sonDone:Z` гарантирует один выбор за процесс.

### Always-on фичи (без тумблера)

Три фичи включены безусловно и не имеют тумблера — это либо приватность по умолчанию, либо системные удобства.

- **🆕 Скриншоты и запись экрана разрешены всегда (Competitor A).** MAX ставит `FLAG_SECURE (0x2000)` на окно в двух местах — блокирует скриншоты и запись экрана. Убраны оба вызова `Window.addFlags(FLAG_SECURE)`: (1) `c1/one/me/android/MainActivity.smali` `:cond_19c` и (2) `c1/v29.smali` `:cond_252`. Скриншоты в приложении работают везде без исключений. Sentinel: `# :allow_screenshots_always`.
- **🆕 IP-адрес не отправляется в телеметрию соединения (Competitor B — no-IP).** В `c2/k10.smali` (сборка map телеметрии соединения) вырезана пара строк, кладущих `"ip"` в map. `connection_type` и `vpn` остаются — они безобидны, IP — самое чувствительное. Всегда включено, тумблер не нужен. Sentinel: `# :noip_telemetry`.
- **🆕 Горизонтальный режим экрана / автоповорот (V6).** `MainActivity.V()` программно вызывал `setRequestedOrientation()`, всегда выбирая portrait независимо от системного автоповорота. Фикс форсирует ветку `SCREEN_ORIENTATION_USER (0x2)` — теперь приложение следует настройке автоповорота устройства (портрет/ландшафт по повороту). `VideoWebViewScreen.setRequestedOrientation(SENSOR/PORTRAIT)` работает независимо (видео вёрстанное фиксированно). Файл: `c1/one/me/android/MainActivity.smali::V`.
- **AppLink prompt для max.ru.** Один раз при запуске MAX показывает диалог «Разрешить открытие ссылок max.ru» (через `DomainVerificationManager`, API 31+) — чинит поведение переподписанного мода, при котором Android не доверяет автоматическому открытию `https://max.ru/*` (что ломало возврат callback Госуслуг «Не нашли чат»). Диалог показывается один раз с `once-guard`, при закрытии сохраняет отметку. Действие: пользователь переходит в системные «Открывать по умолчанию → Открывать поддерживаемые ссылки» и включает `max.ru`. Точка: `MainActivity.onCreate`.

## 1.4 Крэш-фикс — устранён register-collision в `filterPageList`

Диагностика (device-verified S25):

Симптом: в начальных отладочных сборках 26.21.0 при `hideSferum=ON` приложение падало с `IncompatibleClassChangeError: String does not implement List` в `AppFlags.filterPageList` → каскадом в `js6.M`.

Корневая причина — **register-collision в статическом методе `filterPageList(List)List`**. Метод объявлен `.registers 8` при 1 параметре → параметр `p0` (список) занимает старший регистр `v7`. В ветке `hideSferum` выполняется `.toLowerCase(); move-result-object v7` — результат затирает регистр `v7`, то есть саму ссылку на список `p0`. На следующей итерации `List.get(v7 = String)` → runtime-crash.

Исправление: `.registers 9` (параметр смещается на `v8`, `v7` свободен) + defensive-guards `instance-of` на входе (`p0` действительно `List`) и на каждом элементе (действительно `Ljq6`). При Flow type-erasure (или неожиданной эмиссии) метод возвращает список как есть без падения.

Урок для регламента: **`verify-only` (успешный smali-assemble) не ловит runtime-коллизии регистров** — их ловит только запуск на устройстве. Правило регламента `install → logcat → launch → crash-check` из 2026-07-03 зафиксировано в `feedback_smoke_test_regiment.md`. Прошлая версия ICCE в развернутой ранее ветке — **тот же баг** (мисдиагноз о типе `Ljq6`/`Llo6`; тип `Ljq6` верен — `js6` сам кастует к нему в `onBind` стр.140 и `m()` стр.393).

## 2.2 Always-on фичи (без тумблера)
