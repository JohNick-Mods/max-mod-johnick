> Проект создан исключительно в **учебных и исследовательских целях** в области информационной безопасности. Все правки выполнены статическим перепатчем APK и подписаны собственным ключом разработчика. Демонстрирует техническую возможность анализа и нейтрализации модулей телеметрии — **не предназначен для нанесения ущерба правообладателю**.

## Актуальная версия — **26.21.1 V2** (build 6761)

Релиз поверх 26.21.1 V1. Ключевое: **4 новых нейтрализации транспортных серверных рубильников** (TLS-force, DoH-block, WS-redirect-guard, WS-opcode5-dead) + **новый тумблер hideUsefulNotifs** + **тема диалога флагов следует системной**.

- 🌐 **Расширенная защита транспорта — 4 новые нейтрализации.** Сервер MAX может через WebSocket на лету менять «поведение» клиента: включать клиентский DoH (обход DNS-блокировки), отключать TLS-валидацию, перенаправлять на альтернативный WS-узел, включать внутренний LOG-канал. Стоковые дефолты безопасны, но серверная активация происходит без ведома пользователя и без обновления клиента. В V2 закрыты:
  - `net-ssl-session-validate` → форс **TRUE** в парсере `pmsMap` (`c2/lqk.smali::d`). Клиент всегда проверяет сертификат.
  - `net-client-dns-enabled` → форс **FALSE** в том же парсере. DNS-запросы идут в системный резолвер — Lionheart DNS / AdGuard / NextDNS реально применяются.
  - **WS-редирект (`SESSION_INIT.host/port/tls`)** — сервер не может увести клиент на альтернативный узел (guard в `c3/de6.smali`).
  - **WS-опкод 5 (внутренний LOG-канал)** — короткое замыкание в обработчике команд (`c1/zwa.smali::j`), портировано из Geen MAX 26.20.2.
  Все 4 нейтрализации автоматические, без тумблеров — базовая приватность.
- 🔕 **Новый тумблер #16 — `hideUsefulNotifs`.** Скрывает верифицированный промо-чат «Полезные уведомления». Категория «Сервисные чаты» — вместе с `hideVerifyCodes`, `hideMaxOfficial`, `hideInteresting`.
- 🎨 **Тема диалога флагов следует системной.** В V1 при системной тёмной теме диалог настроек мода отображался белым окном. В V2 инициализируется с `Configuration.uiMode`: тёмная система → тёмный диалог, светлая → светлый. Все под-диалоги категорий тоже.
- 💡 **Меню флагов открывается и коротким тапом** по строке версии (не только long-press). Long-press по-прежнему работает.
- ✅ Все функции V1 сохранены — SMS-fix при регистрации (VerGate), hotfix folder-hide через chokepoint `Lu00.b`, скрытый жест long-press → dev-меню MAX, always-on фичи. 16 тумблеров (+1 к V1).
- 📌 Известные ограничения: (1) в режиме Невидимка ручная пометка чата непрочитанным откатывается серверным sync-путём (per-chat реестр с TTL — в следующем релизе); (2) полный `pm clear` клона теряет device_id для push — потребуется повторный вход по SMS.

Сборка универсальная (anti-split — всё в одном APK), работает на всех устройствах:

| Файл | Когда брать |
|---|---|
| `MAX_26.21.1_arm64_V2.apk` | Современные телефоны с 2018 г. **Если не уверены — берите этот.** Samsung, Xiaomi, Realme, Huawei, Pixel… |
| `MAX_26.21.1_arm7_V2.apk` | Старые 32-битные устройства (до 2017 г.) |
| `MAX_26.21.1_arm64_V2_clone.apk` | Клон для второго аккаунта рядом с основным. Оранжевая иконка, пакет `ru.oneme.ap2` |

`SHA256SUMS` — контрольные суммы для проверки.

## Установка

**Обновление с прошлой версии мода:** ставится поверх (подпись та же — cert SHA-1 `d6a7d757…1298b`). **Не удаляйте** предыдущую версию — `install -r` сохраняет данные.

**Первая установка:**
1. Разрешить установку из неизвестных источников (Настройки → Безопасность).
2. Скачать нужный APK из [Releases](../../releases/latest).
3. Открыть APK на телефоне → «Установить».
4. Войти в свой аккаунт (по номеру телефона как обычно).

**Если стоит оригинальный MAX:** удалите перед установкой основного мода (пакет совпадает — `ru.oneme.app`). Клон можно ставить рядом с оригиналом — пакеты разные.

## 16 переключателей в настройках

Короткий **tap** или **long-press** по строке версии (Настройки → О приложении → тап по «Версия ...») откроет диалог с тумблерами. Все по умолчанию **OFF** — до первого включения мод ведёт себя идентично стоку (антидетект). Диалог следует системной теме.

### 🔒 Приватность
- **antiDelete** — антиудалялка (4 слоя + маркер «· удалено»).
- **invisible** — «Невидимка»: typing / read / presence не передаются + режим «Призрак» (сервер видит вас offline).

### 📁 Скрытие папок
- **hideAll** — скрыть папку «Всё».
- **hideSferum** — вкладка «Сферум» + пункт «Войти в Cферум» в настройках.
- **hideFavorites** — скрыть папку и сам чат «Избранное».
- **hideBusinessPromo** — скрыть «MAX для бизнеса» в Settings.
- **hideInviteFriends** — скрыть «Пригласить друзей» в меню.

### 💬 Сервисные чаты
- **hideVerifyCodes** — скрыть чат «Коды подтверждения».
- **hideMaxOfficial** — скрыть чат «MAX» (все 8 омоглиф-комбинаций).
- **hideInteresting** — скрыть чат «Интересное для вас».
- **hideUsefulNotifs** — 🆕 скрыть чат «Полезные уведомления».

### 🧭 Навигация
- **reorderTabs** — «Чаты» первыми вместо «Контактов».
- **startOnNew** — старт на папке «Новые» (основной + клон).

### 🔌 Системное и звонки
- **autoStart** — управление автозапуском в фоне (default OFF).
- **improveNotifications** — восстанавливает WS при смене сети (Wi-Fi ↔ мобильная).
- **backCameraForNote** — видео-кружки с основной камеры вместо фронтальной.
- **debugMenu** — разблокирует скрытое **инженерное меню звонков** (кнопка «Ещё» внутри звонка).

### 📵 Сеть / оператор / профиль
- **blockMobileIdVerify** — скрыть номер от оператора (Header Enrichment / Mobile ID). На «Цифровой ID» **не влияет**.
- **hidePhoneNumber** — скрыть номер в профиле (только отображение).

## Always-on приватность и удобства (без тумблера)

Эти правки включены безусловно.

- **Скриншоты и запись экрана разрешены везде.** Убран `FLAG_SECURE (0x2000)` в двух местах (`MainActivity`, `v29`).
- **IP-адрес не отправляется в телеметрию соединения.** В `c2/k10.smali` вырезана строка `put("ip", …)`; `connection_type` и `vpn` сохраняются (безобидны).
- **Автоповорот экрана следует настройке устройства.** `MainActivity.V()` больше не форсирует portrait — `SCREEN_ORIENTATION_USER`.
- **Диалог «Разрешить ссылки max.ru» (once-guard).** При запуске один раз показывает подсказку — чинит переподписной трюк с App Links для Цифрового ID.
- 🆕 **TLS-валидация не может быть отключена сервером.** Гейт в `pmsMap` парсере форсит `net-ssl-session-validate=true` независимо от серверного значения.
- 🆕 **Клиентский DoH заблокирован.** Гейт в том же парсере форсит `net-client-dns-enabled=false`. DNS-запросы идут в системный резолвер.
- 🆕 **WS-редирект на альтернативный узел заблокирован.** Серверная команда `SESSION_INIT.host/port/tls` игнорируется.
- 🆕 **WS-опкод 5 (внутренний LOG) заглушён** в обработчике команд.

## Антиудалялка — 4 слоя

1. **L3 — Central WS chat-event**. Live-канал, когда собеседник удаляет прямо сейчас и приложение онлайн.
2. **L4 — Room ORM body-clear**. Локальный SQL `UPDATE text=NULL, attaches=NULL, status=10` — блокируется на входе.
3. **V3 persistence (read-layer un-filter)** — SQL-бинды чтения. При `antiDelete=ON` удалённые сообщения остаются видимыми после ре-синка.
4. **Visible marker** — у удалённых сообщений в строке времени дописывается «· удалено».

Общий гейт `antiDelete`: **ON** → защита активна; **OFF** → точное поведение стока. Реакции-эмодзи у соседних сообщений рендерятся корректно.

## Цифровой ID (Госуслуги) — если не открывается

Внутренний фикс встроен в APK — обычно Цифровой ID создаётся сразу. Но из-за переподписи мода Android может не доверять автоматическому открытию ссылок `max.ru`. Лечится один раз:

**Способ 1 — подсказка в приложении.** При запуске MAX сам один раз покажет окно «Цифровой ID» с кнопкой **«Открыть настройки»** → включить «Открывать поддерживаемые ссылки».

**Способ 2 — вручную.** Настройки телефона → **Приложения → MAX → «Открывать по умолчанию» → «Открывать поддерживаемые ссылки»** → включить (`max.ru` должен быть в списке).

**Способ 3 — ADB:**
```bash
# основной
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
# клон
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.ap2 true max.ru
```

Настройка сохраняется при обновлениях мода (`install -r`).

## Что нейтрализовано — 17 / 17

### 🌐 Транспорт (новое в V2)
- **TLS-валидация:** серверный push отключения через `pmsMap["net-ssl-session-validate"]` — форс `TRUE`.
- **Клиентский DoH:** серверный push через `pmsMap["net-client-dns-enabled"]` — форс `FALSE`.
- **WS-редирект (SESSION_INIT.host/port/tls):** серверная переадресация — заблокирована.
- **WS-опкод 5:** служебный канал сериализации трафика — короткое замыкание.

### 🔍 Слежка за устройством и активностью
- **DPS (Data Privacy Stats)** — связка «ID + IP + оператор + VPN-флаг» → `trace-flow.ru`. Stub `DpsInitProvider` + физическое удаление `libtracernative.so`.
- **myTracker (VK/Mail.ru)** — фингерпринт + AD_ID/OAID → `vk-analytics.ru`. Заглушены `initTracker`, `trackEvent`, `flush`.
- **OneLog (call analytics)** — IP/качество звонков → `api.ok.ru`. `UploadService` → `return-void`.
- **Контакты в фоне** — выгрузка адресной книги при foreground (`x10.b()`) заглушена.
- **Загрузка при boot** — гейтится тумблером `autoStart` (OFF по умолчанию).

### 🌐 Сетевые блокировки (NSC)
Операторские Header-Enrichment домены закрыты (`cleartext=false`): `trace-flow.ru`, `sdk-api.apptracer.ru`, `tracker-api.vk-analytics.ru`, `tracker.my.com`, `top-fwz1.mail.ru`.

Гос-эндпоинты (`nuc-cdp.voskhod.ru`, `nuc-cdp.digital.gov.ru`) оставлены доступными — нужны Цифровому ID.

### 📵 Mobile ID / Header Enrichment
Скрытый MSISDN-запрос через оператора — гейт через тумблер `blockMobileIdVerify`.

## Уведомления — если не приходят

MAX получает push через **системные сервисы** (FCM / HMS). Своего канала доставки нет.

1. **Нет GMS/microG → push не придёт никогда**. Поставьте Google Play Services или microG.
2. **Зарубежный трафик режут операторы** → FCM не пробивается. VPN или Wi-Fi без шейпинга.
3. **Производитель усыпляет приложение** (Xiaomi/Huawei/Samsung/Oppo/Vivo):
   - Настройки → Приложения → MAX → **Батарея → Без ограничений**
   - Xiaomi: **Автозапуск** + **Контроль активности → Нет ограничений**
   - Huawei: **Запуск приложений** → выключить «Автоматическое управление»
   - Samsung: Уход за устройством → Батарея → **Ограничения фона → MAX убрать**
4. Включите тумблер **`improveNotifications`** — восстанавливает WS при смене сети.
