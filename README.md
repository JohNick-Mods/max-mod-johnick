Модифицированная сборка мессенджера **MAX** (`ru.oneme.app`) с упором на приватность, нейтрализацию модулей телеметрии и пользовательские UX-функции.

> Проект создан исключительно в **учебных и исследовательских целях** в области информационной безопасности. Все правки выполнены статическим перепатчем APK и подписаны собственным ключом разработчика. Демонстрирует техническую возможность анализа и нейтрализации модулей телеметрии — **не предназначен для нанесения ущерба правообладателю**.

## Актуальная версия — **26.23.2 RS V1** (build 6779)

Первый релиз на новом источнике: базовый APK берётся из **RuStore** (один монолит на все ABI) вместо 4-split бандла Google Play. Итог — **2 универсальных APK** (основной + клон) вместо четырёх. Все фичи прошлой сборки 26.23.0 V8.1 портированы под сток 26.23.2. Ставится через `install -r` поверх любой сборки 26.x — данные сохраняются.

### 🆕 Новое в 26.23.2

- **📦 Источник — RuStore, один универсальный APK.** 2 файла (основной + клон) вместо 4 раздельных arm64/arm7. `x86`/`x86_64` вырезаны ради размера (~50 МБ на APK). Реакции-эмодзи на месте.
- **🗑 Иконка корзины + секунды на удалённых — всегда.** Удалённое сообщение помечается корзиной 🗑, время — с точностью до секунды (`ЧЧ:ММ:СС ДД.ММ`). Без тумблера. Набор «сохранённых удалённых» по-прежнему наполняется при `antiDelete=ON`.
- **📁 «При открытии — папка Новые» работает и в основном приложении.** Раньше `startOnNew` действовал только в клоне (сохранённая папка перетирала выбор) — теперь «Новые» открываются и в основном, и в клоне.
- **🔢 Счётчик непрочитанных не накручивается на холодном старте.** Известное ограничение V8.1 устранено: при «Невидимке» и догнанном чтении бейдж на прочитанных чатах после перезапуска не возвращается; новое входящее показывает бейдж штатно.
- **🎛 Dev-меню без пункта «Дизайн система»** — в стоке это пустой чёрный экран (`stub_release`).
- **🔧 Полный порт под сток 26.23.2** — R8-rebump всех apply-скриптов + полная перекрёстная верификация ~118 патчей.

### ✅ Сохранено из 26.23.0 V8.1

- **🎨 «Цвет иконки» — 6 цветов** (синий/красный/зелёный/фиолетовый/чёрный/оранжевый), меняется на лету без переустановки.
- **📤 Порядок отправленных при «Невидимке»** — пересортировка на входе адаптера (gated `invisible=ON`).
- **🕹 «Невидимка» — единый мастер-тумблер** каскадно управляет `antiRead` / `antiTyping` / `antiPresence`.
- **🛡 Крашфиксы** при запуске (`startOnNew`) и при открытии вкладки «Чаты» (`Resources$NotFoundException`).
- **⌚ Watch-mode** — авто-адаптация плотности под маленькие экраны (умные часы), SMS-код на часах умещается.
- **🌑 AMOLED-тема** — чёрный `#000000` фон, авто при системной тёмной теме.
- **📲 Push у клона стабилен** (собственный `device_id`).
- **🔕 Нет push об удалённых** при `antiDelete=ON`. **⏱→✓** часы на исходящих.
- **🔎 Проверка обновлений** (short-tap) + **автопроверка opt-in** (по умолчанию OFF).

### ⚠ Известные ограничения 26.23.2 RS V1

- **AMOLED** — покрытие ~95%: список чатов и всплывающие шторки могут остаться тёмно-серыми.
- **#FILE-1** — PDF/видео/m4a из основного аккаунта временами отправляются с проблемами. Диагностика продолжается.
- **x86/x86_64** не поддерживаются (вырезаны ради размера) — для эмуляторов x86 нужна отдельная сборка.

### 🤔 Если видишь красный «!» на исходящих или чат открывается не внизу

Проверь тумблер **«Невидимка»** (`invisible`) — это мастер-переключатель, который каскадно взводит `smartAntiRead`, `antiTyping`, `antiPresence`. При `invisible=ON`:
- отметка «прочитано» уходит собеседнику **только после вашего ответа** — собеседник видит непрочитанным, поэтому чат может открываться не в самом низу;
- возможны отдельные ложные «!» на исходящих (когда сообщение реально доставлено, но сервер прислал спорный ответ).

Нужно стандартное поведение — выключи `invisible` и убедись что `smartAntiRead` тоже OFF.

## Установка

**Обновление с прошлой версии мода:** ставится поверх `install -r` (подпись та же). **Не удаляйте** предыдущую версию.

**Первая установка:**
1. Разрешить установку из неизвестных источников.
2. Скачать нужный APK из [Releases](../../releases/latest).
3. Открыть APK → «Установить».
4. Войти по номеру телефона.

**Клон (первая установка):** после установки APK клона — **один раз** Настройки → Приложения → MAX Clone → «Очистить данные» → войти по SMS. Далее не требуется.

## 29 переключателей + выбор цвета иконки

**Long-press** по строке версии (Настройки → О приложении) откроет диалог с тумблерами. **Short-tap** — «Проверить обновления». Все по умолчанию **OFF** (кроме `amoledTheme` — авто при тёмной теме) — до первого включения мод ведёт себя идентично стоку (антидетект). Диалог следует системной теме.

### 🔒 Приватность
- **antiDelete** — антиудалялка (4 слоя + офлайн-персист + маркер 🗑 и время `ЧЧ:ММ:СС`).
- **invisible** — «Невидимка»: мастер-переключатель, взводит дочерние флаги ниже.
  - **antiTyping** — собеседник не видит набор текста.
  - **antiRead** — собеседник не видит отметку «прочитано» + внутричатовый счётчик обнулён + скролл в конец при открытии.
  - **smartAntiRead** — «Умная античиталка»: «прочитано» уходит только **после вашего ответа** (взаимоисключима с обычной).
- **antiPresence** — онлайн / last-seen скрыты (сервер видит offline). **Независимый тумблер**.
- **blockMobileIdVerify** — скрыть номер от оператора (Header Enrichment / Mobile ID). Ломает «Цифровой ID».
- **hidePhoneNumber** — скрыть номер в профиле (только отображение).
- **hideDigitalId** — скрыть пункт «Цифровой ID» из настроек **+ hero-баннер «Госуслуги»**.
- **gateOpcode5** — блокирует принудительный logout от сервера (WS-опкод 5).

### 📁 Папки и вкладки
- **hideAll** — скрыть папку «Всё».
- **hideSferum** — вкладка «Сферум» + пункт «Войти в Сферум» в настройках.
- **hideFavorites** — скрыть папку и чат «Избранное».
- **hideBusinessPromo** — скрыть «MAX для бизнеса» в Settings.
- **hideInviteFriends** — скрыть «Пригласить друзей» в меню.
- **reorderTabs** — «Чаты» первыми вместо «Контактов».
- **startOnNew** — старт на папке «Новые» (в основном и в клоне).

### 💬 Сервисные чаты
- **hideVerifyCodes**, **hideMaxOfficial**, **hideInteresting**, **hideUsefulNotifs**, **hideMaxBusiness** — скрыть соответствующие сервисные чаты.
- **hidePromoWidgets** — скрыть серверные промо-баннеры в списке чатов.
- **hideInviteLinks** — скрыть ссылки «Пригласить по ссылке» / «Найти по номеру».
- **hideChannels** — «Только чаты»: спрятать каналы (главный список + кастомные папки).
- **hideChannelRecs** — скрыть рекомендации каналов.

### 🔌 Системное и звонки
- **autoStart** — управление автозапуском в фоне (default OFF).
- **backCameraForNote** — видео-кружки с основной камеры вместо фронтальной.
- **debugMenu** — разблокирует скрытое инженерное меню звонков.
- **amoledTheme** — чисто чёрный фон `#000000` (окно, status-bar, nav-bar) в тёмной теме. Авто при системной тёмной теме.

### 🎨 Внешний вид
- **iconColor** — цвет иконки приложения: синий / красный / зелёный / фиолетовый / чёрный / оранжевый (radio, меняется на лету).

> Автопроверка обновлений управляется отдельным чекбоксом в окне проверки (не в общем списке). Watch-mode активируется автоматически по ширине экрана — тумблера нет. Тумблер `improveNotifications` присутствует в бинарнике (для двоичной совместимости), но **скрыт из UI** — crash-loop на OxygenOS 15.

## Watch-mode — как работает

Ничего включать не нужно. При каждом старте мод смотрит на ширину экрана в пикселях. Если она **меньше 500 px** (умные часы, компактные носимые устройства) — MAX автоматически переопределяет плотность экрана, из-за чего интерфейс, рассчитанный на телефон, ужимается и умещается на маленьком/круглом экране, не вылезая за края.

На телефонах и планшетах (ширина 1080 px+) режим **не активируется**. Локализация (язык интерфейса) при переключении плотности **сохраняется**.

## «Проверить обновления» — как работает

Настройки → О приложении → **короткий tap** по строке «Версия 26.23.2» → диалог с текущей и последней доступной версией. Запрос идёт в отдельный технический mirror-репозиторий (уменьшенная копия, только для проверки версии). Используется unauthenticated GitHub API — никаких ключей, никакой авторизации. GitHub видит только IP запроса и User-Agent.

**Автопроверка (opt-in):** в этом же окне — чекбокс «Автоматически проверять при запуске (не чаще раза в сутки)». **По умолчанию выключен** — пока не включён, при старте приложения никаких фоновых запросов проверки нет. Включён — раз в 24 часа при запуске идёт тихая проверка; диалог появится только если новая версия реально вышла. Без фоновых будильников/служб.

**Long-tap по версии** по-прежнему открывает дев-меню MAX.

## Антиудалялка — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event** — live-канал, когда собеседник удаляет прямо сейчас.
2. **L4 — Room ORM body-clear** — локальный SQL `UPDATE text=NULL` блокируется.
3. **V3 persistence** — удалённые сообщения остаются видимыми после ре-синка.
4. **Visible marker** — иконка 🗑 и время `ЧЧ:ММ:СС ДД.ММ` в строке времени (безусловно на рендере).
5. **Офлайн-персист (FCM-push)** — удаления, пришедшие пока приложение убито (OEM «Очистить всё» / Force stop / hard-kill), восстанавливаются на следующем старте.

## Цифровой ID (Госуслуги) — если не открывается

Внутренний фикс встроен в APK. Если браузер открывается с «Не нашли чат» — выполните **один раз**:

```bash
# основной
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
# клон
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.ap2 true max.ru
```

Или вручную: Настройки → Приложения → MAX → «Открывать по умолчанию» → «Открывать поддерживаемые ссылки» → включить.

## Что нейтрализовано

- **DPS (Data Privacy Stats)** — `trace-flow.ru`. Stub `DpsInitProvider` + физическое удаление `libtracernative.so`.

- **Tracer-инициализаторы (7×)** и **Firebase Installations** — застаблены (SDK не стартует, нет HTTP к googleapis).
- **Контакты в фоне** — выгрузка адресной книги заглушена.
- **NSC** — операторские Header-Enrichment домены закрыты (`cleartext=false`).
- **WS-опкод 5 (LogoutCmd)** — под тумблер `gateOpcode5`.
