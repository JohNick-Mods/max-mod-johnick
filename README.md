Модифицированная сборка мессенджера **MAX** (`ru.oneme.app`) с упором на приватность, нейтрализацию модулей телеметрии и пользовательские UX-функции.

> Проект создан исключительно в **учебных и исследовательских целях** в области информационной безопасности. Все правки выполнены статическим перепатчем APK и подписаны собственным ключом разработчика. Демонстрирует техническую возможность анализа и нейтрализации модулей телеметрии — **не предназначен для нанесения ущерба правообладателю**.

## Актуальная версия — **26.22.2 GP V1** (build 6779)

**Bump стока 26.22.1 → 26.22.2** — R8-rebump, исправлено 3 бага «Умной античиталки», расширена антидетект-логика на обычный `antiRead`, вернулась «Стартовая папка Новые», добавлена ручная проверка обновлений, реальный чокпоинт баннера «Госуслуги».

> **26.22.2 V1** (2026-07-12): исправлены три бага стока 26.22.2 в «Умной античиталке» (read-mark после ответа, реоткрытие чата = 0 непрочитанных, скролл в конец); обычная `antiRead` теперь тоже без «N новых» и со скроллом; `startOnNew` RE-ENABLED; «Проверить обновления» short-tap по строке версии (mirror-репо, без автопроверки); баннер «Госуслуги» действительно скрыт.

### 🐞 Что исправлено vs 26.22.1 V2

- **🔴 «Умная античиталка» — 3 бага стока 26.22.2 исправлены.** На новом стоке при `smartAntiRead=ON`: (A) метка «прочитано» уходила сразу при открытии чата, а не после ответа; (B) при повторном открытии чата все сообщения снова показывались непрочитанными; (C) чат не скроллился в конец. Все три chokepoint'а стока переехали одновременно — их переязвали заново.
- **🔄 Обычная `antiRead`** теперь тоже блокирует внутричатовый счётчик «N новых» и скроллит в конец. Раньше эта логика работала только при `smartAntiRead`, что нарушало антидетект обычной античиталки.
- **📁 `startOnNew` RE-ENABLED** — «Стартовая папка Новые» вернулась после DEFER в 26.22.0/1.
- **🔎 «Проверить обновления»** — short-tap по строке версии (Настройки → О приложении) открывает диалог с текущей и последней доступной версией. **Автопроверки при старте НЕТ** — только по явному тапу.
- **🇷🇺 Баннер «Госуслуги» — реальный чокпоинт.** В 26.22.1 тумблер `hideDigitalId` прятал только пункт из списка настроек, но не hero-карточку в шапке (она рендерилась другим кодом — PMS-конфиг loop). Теперь скрывается и то, и другое.

### ⚠ Временно отключено в 26.22.2 V1

- **`hideFavorites`** — тумблер в UI есть, но эффекта не даёт. В стоке 26.22.2 рефакторен chat-list combine-оператор (`b20.emit`); нужен fingerprint-resolver моделей. Вернём в 26.22.3+.

## Установка

**Обновление с прошлой версии мода:** ставится поверх `install -r` (подпись та же). **Не удаляйте** предыдущую версию.

**Первая установка:**
1. Разрешить установку из неизвестных источников.
2. Скачать нужный APK из [Releases](../../releases/latest).
3. Открыть APK → «Установить».
4. Войти по номеру телефона.

**Клон (первая установка):** после установки APK клона — **один раз** Настройки → Приложения → MAX Clone → «Очистить данные» → войти по SMS. Далее не требуется.

## 29 переключателей в настройках

**Long-press** по строке версии (Настройки → О приложении) откроет диалог с тумблерами. **Short-tap** — «Проверить обновления». Все по умолчанию **OFF** — до первого включения мод ведёт себя идентично стоку (антидетект). Диалог следует системной теме.

### 🔒 Приватность
- **antiDelete** — антиудалялка (4 слоя + офлайн-персист + маркер «· удалено HH:MM»).
- **invisible** — «Невидимка»: мастер-переключатель, взводит дочерние флаги ниже.
  - **antiTyping** — собеседник не видит набор текста.
  - **antiRead** — собеседник не видит отметку «прочитано» + внутричатовый счётчик обнулён + скролл в конец при открытии.
  - **smartAntiRead** — «Умная античиталка»: «прочитано» уходит только **после вашего ответа** (взаимоисключима с обычной).
  - **antiPresence** — онлайн / last-seen скрыты (сервер видит offline).
- **blockMobileIdVerify** — скрыть номер от оператора (Header Enrichment / Mobile ID). Ломает «Цифровой ID».
- **hidePhoneNumber** — скрыть номер в профиле (только отображение).
- **hideDigitalId** — скрыть пункт «Цифровой ID» из настроек **+ hero-баннер «Госуслуги»**.

### 📁 Папки и вкладки
- **hideAll** — скрыть папку «Всё».
- **hideSferum** — вкладка «Сферум» + пункт «Войти в Сферум» в настройках.
- **hideFavorites** — скрыть папку и чат «Избранное». ⚠️ **DEFER 26.22.2** — временно не работает.
- **hideBusinessPromo** — скрыть «MAX для бизнеса» в Settings.
- **hideInviteFriends** — скрыть «Пригласить друзей» в меню.
- **reorderTabs** — «Чаты» первыми вместо «Контактов».
- **startOnNew** — старт на папке «Новые». ✅ RE-ENABLED.

### 💬 Сервисные чаты
- **hideVerifyCodes**, **hideMaxOfficial**, **hideInteresting**, **hideUsefulNotifs**, **hideMaxBusiness** — скрыть соответствующие сервисные чаты.
- **hidePromoWidgets** — скрыть серверные промо-баннеры в списке чатов (баннер «MAX для бизнеса» с кнопкой «Открыть»).
- **hideInviteLinks** — скрыть ссылки «Пригласить по ссылке» / «Найти по номеру».
- **hideChannels** — «Только чаты»: спрятать каналы (главный список + кастомные папки).
- **hideChannelRecs** — скрыть рекомендации каналов.

### 🔌 Системное и звонки
- **autoStart** — управление автозапуском в фоне (default OFF).
- **backCameraForNote** — видео-кружки с основной камеры вместо фронтальной.
- **debugMenu** — разблокирует скрытое инженерное меню звонков.
- **gateOpcode5** — блокирует принудительный logout от сервера (WS-опкод 5).

> Тумблер `improveNotifications` присутствует в бинарнике (для двоичной совместимости), но **скрыт из UI** — crash-loop на OxygenOS 15. Вернётся в 26.23.x.

## «Проверить обновления» — как работает

**Только по явному тапу.** Автоматических фоновых запросов при старте приложения **НЕТ**.

Настройки → О приложении → **короткий tap** по строке «Версия 26.22.2» → диалог с текущей и последней доступной версией. Запрос идёт в отдельный технический mirror-репозиторий (уменьшенная копия, только для проверки версии). Используется unauthenticated GitHub API — никаких ключей, никакой авторизации. GitHub видит только IP запроса и User-Agent «MaxMod-UpdateChecker/26.22.2».

**Long-tap по версии** по-прежнему открывает дев-меню MAX.

## Антиудалялка — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event** — live-канал, когда собеседник удаляет прямо сейчас.
2. **L4 — Room ORM body-clear** — локальный SQL `UPDATE text=NULL` блокируется.
3. **V3 persistence** — удалённые сообщения остаются видимыми после ре-синка.
4. **Visible marker** — «· удалено HH:MM» в строке времени.
5. **Офлайн-персист (FCM-push)** — удаления, пришедшие пока приложение убито (OEM «Очистить всё» / Force stop / hard-kill), восстанавливаются на следующем старте.

## Цифровой ID (Госуслуги) — если не открывается

Внутренний фикс встроен в APK. Если браузер открывается с «Не нашли чат» — выполните **один раз**:

```bash
# основной
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
# клон
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.ap2 true max.ru
```

Или вручную: Настройки → Приложения → MAX → «Открывать по умолчанию» → «Открывать поддерживаемые ссылки» → включить.
