> Проект создан исключительно в **учебных и исследовательских целях** в области информационной безопасности. Все правки выполнены статическим перепатчем APK и подписаны собственным ключом разработчика. Демонстрирует техническую возможность анализа и нейтрализации модулей телеметрии — **не предназначен для нанесения ущерба правообладателю**.

## Актуальная версия — **26.22.1 V1** (build 6772)

Релиз поверх стока 26.22.1. **29 тумблеров**, все по умолчанию OFF (до первого включения мод ведёт себя идентично стоку — антидетект).

- 🛡️ **Анти-удаление переживает жёсткий kill приложения.** На OEM-прошивках (MIUI/ColorOS/RealMe) системная кнопка «Очистить всё» или «Остановить принудительно» раньше убивали приложение до записи метки — теперь удаления, приходящие через фоновый push, гарантированно попадают в персистентный список. Метка на месте после следующего запуска.
- ⏱️ **Время удаления в метке** — «· удалено HH:MM» вместо просто «· удалено».
- 🇷🇺 **Скрыт промо-баннер «Госуслуги»** на главной странице Настроек (тумблер `hideDigitalId` теперь убирает и баннер, не только пункт списка).
- 📁 **«Только чаты» (hideChannels) работает в кастомных папках** — раньше каналы скрывались только в главном списке, теперь и в пользовательских папках.
- 🔔 **Новый тумблер `improveNotifications`** — фоновая служба уходит на отдельный тихий канал (IMPORTANCE_LOW). Раньше foreground-notif и push-сообщения делили один канал → системное глушение фонового канала глушило и сообщения. При ON — раздельно.
- 🚪 **Новый тумблер `gateOpcode5`** — блокирует принудительный logout от сервера (WS-опкод 5). При ON сессия сохраняется.
- 📱 **Push на клон стабильнее** — устранена ротация device_id при холодных стартах. После установки APK клона нужен один раз `Настройки → Приложения → MAX Clone → Очистить данные`, далее не требуется.
- 📞 **SMS-регистрация нового номера на клоне** — устранена регрессия из промежуточных сборок.
- 🐞 **Багфиксы:** `improveNotifications` больше не роняет клон на Android 12+ (`IMPORTANCE_MIN → LOW`); добавлена NPE-защита в FCM-хендлере анти-удаления.
- ⚠ **Честно:** 2 транспортные нейтрализации из 26.21.1 V2 (WS-redirect guard, `net-ssl-session-validate` / `net-client-dns-enabled`) отложены до 26.23.x. WS-redirect используется штатно для sharding SMS-регистрации; vmk требует передекодировки типов default'ов. Базовые дефолты стока безопасны.
- 📌 Известное ограничение: полный `pm clear` клона теряет device_id для push — потребуется повторный вход по SMS.

## Установка

**Обновление с прошлой версии мода:** ставится поверх (подпись та же — cert SHA-1 `d6a7d757…1298b`). **Не удаляйте** предыдущую версию — `install -r` сохраняет данные.

**Первая установка:**
1. Разрешить установку из неизвестных источников (Настройки → Безопасность).
2. Скачать нужный APK из [Releases](../../releases/latest).
3. Открыть APK на телефоне → «Установить».
4. Войти в свой аккаунт (по номеру телефона как обычно).

**Клон (первая установка):** ставится параллельно с основным модом или оригиналом MAX. После установки APK клона — **один раз** Настройки → Приложения → MAX Clone → «Очистить данные», затем войти. Далее `pm clear` не требуется.

**Если стоит оригинальный MAX:** удалите перед установкой основного мода (пакет совпадает — `ru.oneme.app`). Клон можно ставить рядом с оригиналом — пакеты разные.

## 29 переключателей в настройках

Короткий **tap** или **long-press** по строке версии (Настройки → О приложении → тап по «Версия ...») откроет диалог с тумблерами. Все по умолчанию **OFF** — до первого включения мод ведёт себя идентично стоку (антидетект). Диалог следует системной теме.

### 🔒 Приватность
- **antiDelete** — антиудалялка (4 слоя + офлайн-персист + маркер «· удалено HH:MM»).
- **invisible** — «Невидимка»: мастер-переключатель, взводит дочерние флаги ниже.
  - **antiTyping** — собеседник не видит набор текста.
  - **antiRead** — собеседник не видит отметку «прочитано».
  - **smartAntiRead** — «Умная античиталка»: «прочитано» уходит только **после вашего ответа** в чат (взаимоисключима с обычной античиталкой).
  - **antiPresence** — онлайн / last-seen скрыты (сервер видит offline).
- **blockMobileIdVerify** — скрыть номер от оператора (Header Enrichment / Mobile ID). Ломает «Цифровой ID».
- **hidePhoneNumber** — скрыть номер в профиле (только отображение).
- **hideDigitalId** — скрыть пункт «Цифровой ID» из настроек **+ промо-баннер «Госуслуги»** на главной.

### 📁 Папки и вкладки
- **hideAll** — скрыть папку «Всё».
- **hideSferum** — вкладка «Сферум» + пункт «Войти в Cферум» в настройках.
- **hideFavorites** — скрыть папку и сам чат «Избранное».
- **hideBusinessPromo** — скрыть «MAX для бизнеса» в Settings.
- **hideInviteFriends** — скрыть «Пригласить друзей» в меню.
- **reorderTabs** — «Чаты» первыми вместо «Контактов».
- **startOnNew** — старт на папке «Новые» (основной + клон).

### 💬 Сервисные чаты
- **hideVerifyCodes** — скрыть чат «Коды подтверждения».
- **hideMaxOfficial** — скрыть чат «MAX» (все 8 омоглиф-комбинаций).
- **hideInteresting** — скрыть чат «Интересное для вас».
- **hideUsefulNotifs** — скрыть чат «Полезные уведомления».
- **hideMaxBusiness** — скрыть сервисный чат «Max для бизнеса».
- **hideInviteLinks** — скрыть 2 нижние ссылки «Пригласить по ссылке» / «Найти по номеру».
- **hideChannels** — «Только чаты»: спрятать каналы (главный список + кастомные папки).

### 🔌 Системное и звонки
- **autoStart** — управление автозапуском в фоне (default OFF).
- 🆕 **improveNotifications** — отдельный тихий канал (IMPORTANCE_LOW) для фоновой службы «keep-alive», не глушит push-сообщения при системном DND.
- **backCameraForNote** — видео-кружки с основной камеры вместо фронтальной.
- **debugMenu** — разблокирует скрытое **инженерное меню звонков** (кнопка «Ещё» внутри звонка).
- 🆕 **gateOpcode5** — блокирует принудительный logout от сервера (WS-опкод 5), сохраняя сессию.

## Always-on приватность и удобства (без тумблера)

Эти правки включены безусловно.

- **Отправка любых файлов.** Клиентский блок расширений (`.apk`/`.exe`/…) снят; серверный фильтр по расширению (HTTP 415 `error.extension`) обходится авто-суффиксом **`.bin`** к имени (25 типов). Получатель убирает `.bin` вручную.
- **Изоляция device_id клона.** Клон (`ru.oneme.ap2`) получает собственный device_id → push приходит независимо.
- **Подавление ложного красного «!»** у исходящих в режиме Невидимка (реальные сбои при OFF видны).
- **SMS-авторизация при регистрации работает** — код приходит на новую установку и на клон.
- **Скриншоты и запись экрана разрешены везде.** Убран `FLAG_SECURE (0x2000)`.
- **IP-адрес не отправляется в телеметрию соединения.** В `c2/k10.smali` вырезана строка `put("ip", …)`.
- **Автоповорот экрана следует настройке устройства.**
- **Диалог «Разрешить ссылки max.ru» (once-guard).** Чинит переподписной трюк с App Links для Цифрового ID.

> ⚠ **Регресс относительно 26.21.1 V2:** 2 транспортные нейтрализации (`net-ssl-session-validate=true`, `net-client-dns-enabled=false`, WS-redirect guard) в 26.22.1 V1 отложены до 26.23.x. Причины: WS-redirect используется штатно для sharding SMS-регистрации (безусловный skip ломает вход); vmk-реестр `PmsProperty` изменил тип default'ов (тривиальная замена → ClassCast). Дефолты стока безопасны. **WS-опкод 5** — вернулся под тумблер `gateOpcode5` в этой версии.

## Антиудалялка — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event**. Live-канал, когда собеседник удаляет прямо сейчас и приложение онлайн.
2. **L4 — Room ORM body-clear**. Локальный SQL `UPDATE text=NULL, attaches=NULL, status=10` — блокируется на входе.
3. **V3 persistence (read-layer un-filter)** — SQL-бинды чтения. При `antiDelete=ON` удалённые сообщения остаются видимыми после ре-синка.
4. **Visible marker** — у удалённых сообщений в строке времени дописывается «· удалено HH:MM».
5. 🆕 **Офлайн-персист (FCM-путь)** — удаления, пришедшие пока приложение убито OEM-killer'ом, восстанавливаются на следующем старте (метка выживает Force Stop / «Очистить всё»).

Общий гейт `antiDelete`: **ON** → защита активна; **OFF** → точное поведение стока. Реакции-эмодзи у соседних сообщений рендерятся корректно.

## Цифровой ID (Госуслуги) — если не открывается

Внутренний фикс встроен в APK — обычно Цифровой ID создаётся сразу. Но из-за переподписи мода Android может не доверять автоматическому открытию ссылок `max.ru`. Лечится один раз:

**Способ 1 — подсказка в приложении.** При запуске MAX сам один раз покажет окно «Цифровой ID» с кнопкой **«Открыть настройки»** → включить «Открывать поддерживаемые ссылки».

**Способ 2 — вручную.** Настройки телефона → **Приложения → MAX → «Открывать по умолчанию» → «Открывать поддерживаемые ссылки»** → включить (`max.ru` должен быть в списке).

**Способ 3 — ADB:**
```bash
# основной
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
# клон
adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.ap2 true max.ru
```

Настройка сохраняется при обновлениях мода (`install -r`).

### 🔍 Слежка за устройством и активностью

- **myTracker (VK/Mail.ru)** — фингерпринт + AD_ID/OAID → `vk-analytics.ru`. Заглушены `initTracker`, `trackEvent`, `flush`.

- **Контакты в фоне** — выгрузка адресной книги при foreground заглушена.

Операторские Header-Enrichment домены закрыты (`cleartext=false`): `trace-flow.ru`, `sdk-api.apptracer.ru`, `tracker-api.vk-analytics.ru`, `tracker.my.com`, `top-fwz1.mail.ru`.

## Уведомления — если не приходят

MAX получает push через **системные сервисы** (FCM / HMS). Своего канала доставки нет.

1. **Нет GMS/microG → push не придёт никогда**. Поставьте Google Play Services или microG.
2. **Зарубежный трафик режут операторы** → FCM не пробивается. VPN или Wi-Fi без шейпинга.
3. **Производитель усыпляет приложение** (Xiaomi/Huawei/Samsung/Oppo/Vivo):
   - Настройки → Приложения → MAX → **Батарея → Без ограничений**
   - Xiaomi: **Автозапуск** + **Контроль активности → Нет ограничений**
   - Huawei: **Запуск приложений** → выключить «Автоматическое управление»
   - Samsung: Уход за устройством → Батарея → **Ограничения фона → MAX убрать**
4. Включите тумблер **`improveNotifications`** — уводит фоновую службу на отдельный тихий канал (IMPORTANCE_LOW), не глушится вместе с DND.
