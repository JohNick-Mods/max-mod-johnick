## 🆕 Новое и исправленное в 26.23.0 V2

### 🔎 Исправлена проверка обновлений (ложное «доступно обновление»)

**Симптом**: на опубликованной V1 короткий тап по версии показывал «доступно обновление», даже когда установлена самая свежая сборка.

**Корневая причина**: метод `currentVersion()`, с которым `UpdateChecker` сравнивает тег mirror-репозитория, содержал устаревшую жёстко-зашитую строку версии — она не обновлялась при пересборке. Из-за этого при равных версиях мод считал mirror-тег «новее».

**Фикс**: значение версии для сравнения вынесено в единый источник (`build_config.py`) и **пересобирается из него при каждой сборке** — рассинхрон устранён на корню. При актуальной установленной версии проверка теперь корректно отвечает «Обновлений не обнаружено».

### 🔔 Автопроверка обновлений — новый opt-in чекбокс (по умолчанию OFF)

В диалоге проверки обновлений (short-tap по версии) добавлен чекбокс **«Автоматически проверять при запуске (не чаще раза в сутки)»**.

- **По умолчанию выключен.** Пока не включён — приложение при старте **не делает ни одного сетевого запроса** проверки обновлений (поведение идентично V1).
- При включении: раз в 24 часа (скользящее окно) при старте приложения выполняется **тихая** проверка. Диалог показывается **только если новая версия действительно есть** — при её отсутствии или сетевой ошибке пользователь ничего не видит.
- Без фоновых планировщиков (`AlarmManager` / `WorkManager` / `JobScheduler`) — только однократная inline-проверка при запуске под двойным гейтом «флаг включён + прошли сутки».
- Хранение флага и метки последней проверки — в отдельном `SharedPreferences` «app_flags»; до первого включения запись отсутствует.

> Управление — только этим чекбоксом; в общем списке тумблеров пункт скрыт, чтобы не дублировать.

### ⌚ Watch-mode временно отключён

Авто-адаптация под смарт-часы (была заявлена в V1) **отключена в V2**. По результатам ревью сборочного пайплайна выяснилось, что блок watch-mode использовал регистры, недопустимые для соответствующих smali-инструкций, и фактически **не попадал в собранный APK ещё в V1** (на телефонах режим не активируется, поэтому дефект был незаметен). Чтобы не держать неработающий код, режим убран из пайплайна. Полноценная реализация запланирована на следующую версию.

## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) — независимый тумблер |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно. Оба блокируют внутричатовый счётчик «N новых» и скроллят в конец при открытии.

### Анти-удаление (`antiDelete`) — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event**: live-канал, собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear**: локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер «· удалено»** в строке времени + **время удаления** (`dk_times`).
5. **Офлайн-удаления через FCM** — метка восстанавливается при следующем старте, если удаление пришло при убитом OEM-killer'ом приложении.

Восстановленные удалённые сохраняются с `isRead=1` — не задирают счётчик непрочитанных. Общий гейт `antiDelete`: **ON** → защита; **OFF** → поведение стока.

### 📢 Каналы = сток при включённых античиталках
При `antiRead`/`smartAntiRead`/`invisible` каналы ведут себя как в стоке: read-mark для каналов всегда уходит на сервер.

### 🔕 Приваси-стабы (P0/P1)
- **myTracker (VK)** — вызовы SDK нейтрализованы (`return-void`).
- **Фоновая выгрузка контактов** — метод нейтрализован (`return-void`).

## ⚠ Честно: чего НЕТ / ограничения в 26.23.0 V2

- **Watch-mode временно отключён** — некорректно собирался; полноценная реализация перенесена на следующую версию.
- **AMOLED — покрытие ~95%**: список чатов, окно чата и bottom-sheets могут остаться тёмно-серыми.
- **`#FILE-1`**: PDF / видео / m4a из основного аккаунта временами отправляются с проблемами — диагностика продолжается.
- **Тумблер `improveNotifications` временно скрыт из UI** (жёсткие проверки Android 15 / OxygenOS 15).
- **2 транспортные нейтрализации остаются DEFERRED**:
  - `WS-redirect` — используется штатно для sharding SMS-регистрации; безусловный skip ломает вход по новым номерам.
  - `vmk` (SSL-session-validate / DNS-client-config) — реестр `PmsProperty` изменил тип default'ов.
- **«Цифровой ID» и «Госуслуги баннер»** контролируются одним `hideDigitalId` (общий серверный ключ PmsProperty).
- **Требуется one-time `pm clear ru.oneme.ap2`** после установки APK клона.
- **Минимальная версия Android — 8.0** (API 26) — ограничение стока.
