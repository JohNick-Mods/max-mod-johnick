# MAX 26.22.1 AG V1 — Технический аудит мода

**Версия стока**: 26.22.1 (build 6772), сборка **AppGallery / Huawei**
**Версия мода**: V1 (Privacy Mod by JohNick, AG-редакция)
**Дата**: 2026-07-11
**Совместимость**: `ru.oneme.app` (Android 8.0–16.0 / API 26–36) · arm64-v8a + armeabi-v7a · основной + клон (`ru.oneme.ap2`)

> **AG V1** — первый релиз мода на основе AppGallery-сборки MAX (Huawei-канал), портированный с рабочей GP V3. Все 43 активных патча GP V3 перенесены и адаптированы под R8-раскладку AG. Полностью совместим с устройствами без Google Play Services (Huawei HarmonyOS, Honor MagicOS и др.) — работает через HMS-стек уведомлений.

---

## 🆕 Что нового в AG-редакции

### Работа без Google Play Services (HMS-стек)
В AppGallery-сборке MAX Firebase Cloud Messaging заменён на **Huawei Push Kit (HMS)**. GP-версия мода на устройствах Huawei новее 2020 г. (без GMS) push-уведомления не получала. **AG V1** использует нативный HMS-стек стока — уведомления работают на всём парке Huawei/Honor без Google-сервисов.

### Fat-APK разделён на thin по ABI
AppGallery-исходник — это единый fat-APK (~148 MB) со всеми ABI одновременно. Мод режет его на два раздельных universal APK (arm64 + arm7), каждый ~40 MB — как и в GP-редакции. Ниже вес, ниже трафик обновлений, без x86.

### Все GP-фичи, портированные под AG
Полный набор из V1 26.22.1 + V2 26.22.1 + V3 26.22.1 GP работает и здесь — с точечными адаптациями под другую R8-раскладку и структуру DEX (в AG **4 DEX** вместо 3 в GP).

## 🐞 Исправления, унаследованные от GP-веток

### Fix «Умной античиталки»: открытие диалога прыгает к последнему сообщению *(порт V3 GP)*
При включённой «Умной античиталке» (`smartAntiRead`) сервер не получает read-mark → продолжает считать все сообщения непрочитанными → при открытии диалога клиент вычислял позицию по «первому непрочитанному» и вставал туда, а не в самый низ.

**Фикс (AG-версия)**: инжект в **точку создания scroll-события** (`Lece.e()`) — при `smartAntiRead=ON` mark принудительно заменяется на `Long.MIN_VALUE`, что уводит `scroll-executor` в специальную ветку стока, вызывающую `RecyclerView.smoothScrollToPosition(0)` — в reverse-layout адаптере чата это последнее сообщение (низ экрана). Условие срабатывает только для initial-open — переход по клику на цитату / из поиска использует другие точки создания события (`Lece.i()` / `Lece.j()`) — не тронуты.

*Отличие от GP V3*: GP патчил `Lyca.c()` (executor), AG-версия — `Lece.e()` (создатель события). В GP тот инжект тоже работал бы, но в AG метод-executor (`Lv9a.c()`) переиспользует один регистр с 8+ разными типами, что делает Java-инжект в него невозможным без ART VerifyError. Обходной путь через `Lece.e()` семантически чище — использует уже существующую в стоке «нет непрочит → в конец» ветку.

При выключенном `smartAntiRead` поведение = стоку.

### Crash-loop `improveNotifications` (OxygenOS 15) *(V2 GP → AG)*
Тумблер **скрыт из UI** (не появляется в диалоге); шаг создания отдельного канала `.bg` **отключён** в пайплайне сборки. `BackgroundListenService` работает на стоковом канале `.misc` — крашей нет. Поле `improveNotifications:Z` в `AppFlags` сохранено — для двоичной совместимости с классами `KeepAliveManager` / `NetworkChangeCallback`.

### `hidePromoWidgets` — скрытие серверных промо-баннеров в списке чатов *(V2 GP → AG)*
Тумблер (категория «Чаты», OFF-default) — удаляет ключ `chats-list-promo-link-enabled` из PMS-конфига при инициализации `Ljh4.<init>(Map)` (в GP то же место было `Lxs5`). Getter возвращает null → баннер «MAX для бизнеса» с кнопкой «Открыть» между обычными чатами не рендерится.

### V1.2d hotfix (AG-специфичный): Turkish-locale fix в фильтре настроек
Матчер серверных пунктов в `filterSettingsList` использовал `String.toLowerCase()` без локали. На устройствах с турецкой локалью буква `I` мапится на dotless `ı`, из-за чего матчер `"max id"` не срабатывал бы (`"MAX ID"` → `"max ıd"`). Заменено на `toLowerCase(Locale.ROOT)` — стабильно на любой локали.

## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа; при открытии диалога чат сразу на последнем сообщении |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно.

### Анти-удаление (`antiDelete`) — 4 слоя + офлайн-персист

1. **L3 — Central WS chat-event**: live-канал, собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear**: локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер «· удалено»** в строке времени + **время удаления**.
5. **Офлайн-удаления через HMS-push** — если удаление пришло, пока приложение убито OEM-killer'ом, метка восстанавливается при следующем старте (msgId заносится в персистентный список на входе push `MessageRemoved`).

### Скрытие сервисных чатов и промо
`hideVerifyCodes`, `hideMaxOfficial`, `hideInteresting`, `hideUsefulNotifs`, `hideMaxBusiness`, `hidePromoWidgets`, `hideInviteLinks`, `hideChannels`, `hideChannelRecs`.

### Скрытие пунктов настроек (server-driven)
`hideSferum`, `hideDigitalId`, `hideBusinessPromo`, `hideInviteFriends` — фильтруют server-driven строки настроек по res-id (`0x7f110aba`, `0x7f110a9c`) и по тексту (`«ферум»`, `«цифров»`, `«max id»`, `«госуслуг»`) с учётом Locale.ROOT.

## ⚠ Известные ограничения AG V1

### Баннер «госуслуги» на экране Настроек — НЕ скрыт
На экране Настроек между профилем и первым пунктом меню отображается широкий баннер с брендированным лого «госуслуги». В GP V1 этот баннер прячется через тумблер `hideDigitalId`. **В AG-сборке** этот баннер живёт как отдельный view-элемент в бинарном layout XML с default drawable-анимацией (`$max_id_avd`), а не через Java-контроллер `Lga5`, который был точкой гейта в GP. Три попытки перехвата (`hide_digital_id_banner` в Lga5.k, skip new-instance Lga5 в MainScreen, findViewById(max_id_container).setVisibility(GONE)) не убирают баннер — рендер идёт из другого источника, которого точечный recon пока не нашёл.

**Обход:** пункт «Цифровой ID» в списке настроек скрывается корректно (через `filterSettingsList`); функциональность Госуслуг/ЕСИА не блокируется (при желании — можно кликнуть баннер и войти как обычно).

Фикс баннера — приоритет для AG V1.1 hotfix после дополнительного recon.

## 📝 Отличия от GP V3

- **Push**: HMS вместо FCM (нативный path AG-стока).
- **DEX**: 4 файла вместо 3.
- **R8-раскладка**: полностью иная (полный rebump 87 хардкод-якорей).
- **Fat-APK → thin**: обратная операция vs GP (там merge splits, здесь split fat).
- **Fix античиталки**: другая точка гейта (`Lece.e()` вместо `Lyca.c()`) — более чистая через использование стоковой ветки.
- **Известное ограничение**: баннер «госуслуги» на Настройках не убирается (в GP убирается).
