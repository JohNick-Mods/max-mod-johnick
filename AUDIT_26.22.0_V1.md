## 🛡 Профиль приватности

### Невидимка (`invisible`) — мастер-переключатель
Взводит дочерние приватные флаги; каждый работает и отдельно.

| Дочерний флаг | Что делает |
|---|---|
| Антипечатанье (`antiTyping`) | собеседник не видит набор текста |
| Античиталка (`antiRead`) | собеседник не видит отметку «прочитано» |
| Умная античиталка (`smartAntiRead`) | «прочитано» уходит только ПОСЛЕ вашего ответа |
| Антиприсутствие (`antiPresence`) | онлайн / last-seen скрыты (сервер видит offline) |

**Взаимоисключение**: `antiRead` и `smartAntiRead` не могут быть включены одновременно — включение одного снимает другой.

### 🆕 Умная античиталка (`smartAntiRead`)
Отметка «прочитано» отправляется собеседнику **только после того, как вы сами ответили** в этот чат. До ответа — сообщения читаются молча (собеседник не видит галочку). Реализация:
- **latch** — при отправке сообщения chatId+serverId помещаются в реестр «отвеченных» чатов (`fla.f`, резолв serverId через `tp2.N`).
- **gate** — отправка read-mark подавляется хелпером `shouldSuppressSmartRead` (в `wjd.c`), пока чат не в реестре.
- **счётчик непрочитанных** обнуляется при активном гейте и в списке чатов (`oa3`→`o03`), и внутри открытого чата (плашка «N новых» — `oca.f`).

### Анти-удаление (`antiDelete`) — 4 слоя
1. **L3 — Central WS chat-event** (`y8b.b`): live-канал, собеседник удаляет при онлайн-приложении.
2. **L4 — Room ORM body-clear** (`r3e.z` / `zhe.A`): локальный SQL-сброс тела блокируется.
3. **V3 persist**: удалённые сообщения остаются видимыми после ре-синка.
4. **Маркер «· удалено»**: в строке времени.

Общий гейт `antiDelete`: **ON** → защита; **OFF** → поведение стока.

## 🆕 Новое в 26.22.0 (относительно 26.21.1)

- **Умная античиталка** (`smartAntiRead`) — read-mark после ответа + сброс счётчика непрочитанных (список и внутри чата).
- **Невидимка-каскад** — `invisible` взводит умную античиталку и остальные приватные флаги.
- **Скрыть «Max для бизнеса»** (`hideMaxBusiness`) — сервисный чат.
- **Скрыть «Пригласить/Найти по номеру»** (`hideInviteLinks`) — 2 нижние гиперссылки в списке чатов.
- **«Только чаты»** (`hideChannels`) — скрыть каналы, оставить личные диалоги и группы.
- **Отправка любых файлов** (always-on) — `.apk`, `.exe` и др. опасные типы. Клиентский блэклист расширений снят; серверный фильтр (HTTP 415 `error.extension`) обходится авто-суффиксом **`.bin`** к имени файла (25 расширений: exe/msi/apk/dmg/app/ipa/com/pif/scr/bat/cmd/vbs/js/sh/py/pl/ps1/jar/reg/sys/dll/lnk/docm/xlsm/pptm). Получатель убирает `.bin` вручную.
- **Клон-push фикс** — push на клон (`ru.oneme.ap2`) больше не зависит от `pm clear`: device_id изолируется на LOCAL- и REMOTE-источниках (клон получает свой UUID).
- **Красный «!» фикс** — убран ложный ERROR-статус исходящих при активной Невидимке (реальные сбои при OFF по-прежнему видны).
- **Сброс отката «Отметить непрочитанным»** в Невидимке (read-ledger, устраняет known-issue из 26.21.1).

## 🔓 Always-on (без тумблера)

- Отправка любых файлов (обход клиентского блэклиста + маскировка `.bin` для сервера).
- Изоляция device_id клона (LOCAL + REMOTE → свой push).
- Подавление ложного красного «!» при Невидимке.
- SMS-авторизация при регистрации работает (клиент представляется серверу совместимой версией).
- Скриншоты и запись экрана разрешены везде (`FLAG_SECURE` снят).
- IP-адрес не отправляется в телеметрию соединения (`connection_type` / `vpn` сохранены).
- Автоповорот следует настройке устройства.
- Диалог «Разрешить ссылки max.ru» (one-guard) — чинит App Links для Цифрового ID.
- TrustManager accept-all → отклоняется; logcat-dump → выключен; фоновая выгрузка контактов → выключена; NFC-HCE → SW 6F00; myTracker (VK) → выключен.
- Антиудаление (4 слоя), tracer-stubs, AppFlags init, KeepAlive, dev-меню MAX по long-press, dynamic FileProvider authority для клона.

## ⚠ Честно: чего НЕТ в 26.22.0 V1

Относительно 26.21.1 V2 **не активны 4 транспортные нейтрализации** (форс `net-ssl-session-validate=true`, `net-client-dns-enabled=false`, WS-redirect guard, WS-опкод 5): в стоке 26.22.0 их якоря (`vmk`/`de6`/`zwa`/`lqk`) переименованы R8 и требуют повторного recon + device-smoke. Планируется вернуть в следующем обновлении. Базовые дефолты стока в этих точках безопасны.

**Известные ограничения**: полный `pm clear` клона теряет device_id для push — потребуется повторный вход по SMS.

### 🐞 Исправлено в этой сборке

- **Отправка сообщений** — умная античиталка резолвила serverId в hot-path отправки и роняла путь в NPE (сообщения зависали с «часами»). Резолв перенесён в надёжную точку создания задачи отправки (`ave.D`), где serverId гарантированно доступен; отметка «прочитано» после ответа сохранена.
- **Реакции** — восстановление потерянных VK anti-split'ом emoji-спрайтов в `resources.arsc` использовало сдвинутый на −1 диапазон resource ID (константы от прошлой версии стока) → в пикере отображался один эмодзи, а ставился соседний. Диапазон измерен из density-split стока 26.22.0 и выровнен (`emoji_sprite_0` = `0x7f0804ae`).
- **Скрытие «Пригласить/Найти»** — флаг гасил только popup-меню на экране Контактов; теперь фильтруется и футер списка чатов во вкладке «Все» (реальная точка рендера).
