# MAX 26.21.0 — Mods by JohNick V1

Универсальная сборка (GP) · arm64 + arm7 · Main + Clone · 4 APK · build 6761

## Что нового (vs 26.20.2)

- **🆕 Перенос на свежий сток 26.21.0 (build 6761).** R8-имена сверены по содержимому, все патчи портированы и проверены регрессией + smoke S25. Все 13/13 угроз аудита остаются нейтрализованы.
- **🆕 15-й тумблер:** `hideFavorites` — скрыть папку **«Избранное»** в списке чатов. По умолчанию OFF.
- **✅ `hideSferum` — теперь скрывается и вкладка в полоске, и пункт «Войти в Cферум» в настройках.** В 26.20.2 фильтр стоял только в адаптере страниц (страницы прятались, полоска — нет) и на промежуточном addAll в списке настроек (пункт «Cферум» не ловился). Теперь фильтр вкладок перенесён в источник (до tab-strip и pages), фильтр настроек — на финальную эмиссию списка (ловит поздние вставки).
- **✅ `hidePhoneNumber` — правильная точка для нового стока.** Перенесён с мёртвой в 26.21.0 точки `wb4.q()` на реальный геттер `Ld74.w()J` — покрывает 40+ callsites отображения. Смежное поле `serverId`/`userId` сознательно не трогается (используется 20+ core-классами внутренней маршрутизации).
- **✅ `startOnNew` — реактивирован под новый folder-стек.** В 26.21.0 подсистема папок переписана — точка хука перенесена в `rf3.pswitch_2b5` после привязки списка к UI, выбор через прямой `setCurrentItem` у ViewPager2. Работает и в основном приложении, и в клоне.
- **🔧 Крэш-фикс filterPageList.** Начальные отладочные сборки падали с `IncompatibleClassChangeError` при `hideSferum=ON` — register-collision в статическом методе (параметр-список затирался результатом `toLowerCase()`). Исправлено расширением количества регистров + defensive instance-of guards. Урок: `verify-only` не ловит runtime-коллизии — правило регламента «install → logcat → launch → crash-check» встроено в каждую сборку.

## Установка

**Обновление с любой 26.x от Mods by JohNick:** ставится поверх, данные сохраняются. Cert SHA-1 `d6a7d757…1298b` — одинаков. **Не удаляйте** предыдущую версию.

**Первая установка:** разрешить установку из неизвестных источников → открыть APK → «Установить» → войти по номеру.

**Если стоит оригинальный MAX:** удалить перед установкой основного мода (пакет совпадает). Клон можно ставить рядом с оригиналом.

## Цифровой ID — если открывается браузер

Внутренний фикс из 26.20.2 встроен — обычно Цифровой ID создаётся сразу. Из-за переподписи Android может не доверять автоматическому открытию ссылок `max.ru`. Лечится один раз:

- В приложении: попап «Цифровой ID» → «Открыть настройки» → «Открывать поддерживаемые ссылки»
- Или: **Настройки телефона → Приложения → MAX → «Открывать по умолчанию» → включить**
- Или через ADB:
  ```
  adb shell pm set-app-links-user-selection --user 0 --package ru.oneme.app true max.ru
  ```

Подробности — `DIGITAL_ID_README.md`.
