# ЧАСТЬ 1. Что нового относительно 26.21.0

## 1.1 Платформа

Мод собран на том же стоке **MAX 26.21.0 (build 6761)**, что и предыдущий релиз — R8-карта не менялась, VersionName обновлён до `26.21.1` для install-r поверх любой версии. Все нейтрализации и функции 26.21.0 сохранены без потерь.

## 1.2 Hotfix — скрытие чатов «Избранное» / «Коды подтверждения» / «MAX» в списке чатов

В 26.21.0 фильтры `hideFavorites` / `hideVerifyCodes` / `hideMaxOfficial` работали через `filterPageList` — это скрывало соответствующие **папки-вкладки** в верхней полоске. Но сами чаты «Избранное», «Коды подтверждения» и официальный «MAX» продолжали отображаться в списке чатов на вкладке «Все». Тумблеры визуально «не работали».

Расследование выявило три ортогональных причины:

**1. Истинный chokepoint AsyncListDiffer — `Lu00.b(List, Runnable)V`, а не `Ljq8.H/I`.** Класс `Ljq8` — базовый `ListAdapter`, из которого наследуются все чат-адаптеры, — умеет два публичных метода submit: `H(List)V` и `I(List, Runnable)V`. Оба делегируют внутрь на `Lu00.b`. Однако около десяти direct-callsites (`gt6`, `jt6`, `se9`, `sx2`, `qkg`, `vm5` и другие) вызывают `Lu00.b` **напрямую**, обходя обёртки `Ljq8`. Поэтому инъекция фильтра в `Ljq8.H/I` пропускает большую часть визуальных путей. В 26.21.1 фильтр помещён в сам `Lu00.b` — это единственный универсальный chokepoint, покрывающий все пути AsyncListDiffer.submitList независимо от адаптера-инициатора. Sentinel: `# :u00_submit_filter`. Скрипт: `apply_v75_u00_submit_filter.py`.

**2. Модель UI recent-chats — `Lf03`, а не `Ly66` и не `Lmm2`.** Прежний фильтр умел обрабатывать элементы модели `Ly66` (master-index чатов, большой список из сотен элементов, эмитится глубже в WS-цепочке). Runtime-диагностика показала, что визуальный список из 6–8 recent-chats состоит из объектов другого класса — `Lf03`. Это отдельная UI-модель с полями `a:J` (chatId), `b:Uri` (icon), `c:CharSequence` (title). Поэтому `filterChatList` дополнен ветвью для `Lf03`: `instance-of Ly66 → title=e:CharSequence` (эмит-путь) / `instance-of Lf03 → title=c:CharSequence` (UI-путь).

**3. Официальный чат «MAX» использует смешанный омоглиф-набор (Latin/Cyrillic).** Буквы `M`, `A`, `X` внешне идентичны кириллическим `М`, `А`, `Х` — сервер может отдавать любую из 2³=8 комбинаций. Строгий `equals("max")` в lowercase покрывал только один вариант. Фильтр перебирает все восемь: `max`, `maх`, `mаx`, `mах`, `мax`, `мaх`, `маx`, `мах`. Точность `equals` остаётся высокой — пользовательские чаты не называются ровно из трёх букв в этом наборе.

**Побочное решение — снят verified-guard.** В прошлой версии ветка `hideMaxOfficial` дополнительно требовала boolean-поле `Ly66.d:Z` (флаг verified-badge) = `true`, чтобы исключить ложные совпадения с пользовательскими чатами. Для модели `Lf03` соответствующий boolean `l:Z` — **не verified** (какой-то другой признак), а другого явного verified-поля в `Lf03` не обнаружено. В 26.21.1 guard снят полностью — точность обеспечивает строгое `equals` по 8 омоглиф-комбинациям. Практически: если пользователь сам назовёт свой чат ровно `MAX`/`МАХ` в любом омоглиф-миксе, он тоже скроется при включённом тумблере. Считаем приемлемым: имя ровно из трёх букв в этом наборе на практике не встречается, а surface изменения — только UI-фильтрация списка (данные не удаляются, поиск работает).

**Результат (device-verified S25):** при включённых тумблерах Избранное / Коды подтверждения / MAX исчезают из списка чатов на вкладке «Все». Наблюдения — стабильны через install-r, force-stop, свап-между-вкладок.

**Область изменений — только UI-фильтрация списка чатов.** Никакие данные не удаляются на устройстве и не отправляются на сервер; изменения не затрагивают сеть, авторизацию, шифрование и разрешения. Скрытые чаты остаются доступны через поиск (глобальный поиск по имени/содержимому).

## 1.3 Изменения в безопасности относительно 26.21.0

Изменений в наборе нейтрализованных угроз нет. 13/13 угроз аудита 26.21.0 остаются нейтрализованы теми же классами и по тем же точкам патча. Стоковый build, `network_security_config`, permissions, набор нативных библиотек — идентичны 26.21.0.

## 1.4 Известные ограничения (не покрыто в 26.21.1)

- **«Отметить непрочитанным» в режиме Невидимка (antiRead ON).** Ручная пометка чата непрочитанным через контекстное меню откатывается серверным sync-путём (не только READ-side гейт `V71`, но и WRITE-side `azh.smali:150`). Исправление требует отдельного per-chat реестра `{chatId: timestamp}` с TTL~5 мин — архитектурная работа, вынесена на будущий релиз.

## 1.5 Метрики

- **15** пользовательских переключателей (без изменений относительно 26.21.0).
- Регрессия: pytest прошла на уровне 26.21.0.
- verify критических слоёв — **0 FATAL**; smali-сборка 3 DEX без VerifyError; 4 APK подписаны (apksigner v2+v3, без v1).
- Smoke на устройстве — **PASS** (main + clone запускаются без FATAL/VerifyError/IncompatibleClass; функциональная проверка — Избранное / Коды подтверждения / MAX действительно скрываются из списка чатов).

# ЧАСТЬ 2. Функции (перечень без изменений относительно 26.21.0)

15 переключателей + скрытый жест long-press по номеру версии + always-on фичи (скриншоты, no-IP telemetry, автоповорот, AppLink prompt). Полное описание — в аудите 26.21.0.

| 1 | Антиудаление | Удалённые собеседником сообщения остаются с пометкой «· удалено» |

| 3 | Скрыть папку «Всё» | Убирает вкладку-папку «Всё» в списке чатов |
| 4 | Скрыть папку «Сферум» | Убирает вкладку «Сферум» + пункт «Войти в Cферум» в настройках |
| 5 | Скрыть папку «Избранное» | Убирает вкладку-папку **и сам чат «Избранное»** в списке чатов *(в 26.21.1 hotfix — скрывает и чат)* |
| 6 | Скрыть «MAX для бизнеса» | Убирает рекламную карточку в настройках |

| 10 | Автозапуск MAX | Управляет автозапуском после перезагрузки |
